# Web 端手机版聊天头像进入好友详情页规划

## 任务目标
- 打通 `Web` 手机版单聊页面里“点击对方头像 -> 进入好友信息详情页”的主链路。
- 将现有 `apps/app/src/routes/character-detail-page.tsx` 的移动端视图改造成更接近微信手机版“朋友信息”页面的结构和视觉。
- 本次不新增后端接口，不新增测试文件，不扩展新路由，优先复用已有资料接口与 `/character/$characterId` 路由。

## 改造范围
- `apps/app/src/components/chat-message-list.tsx`
- `apps/app/src/routes/character-detail-page.tsx`

## 当前问题
- 移动端聊天消息里的头像当前仅作展示，只有桌面端头像支持进一步操作。
- 现有 `character-detail-page.tsx` 移动端版本仍偏“角色资料页”，顶部操作、信息分组和底部主按钮都不像微信手机版的“朋友信息”页面。

## 目标交互

### 聊天页
- 在移动端单聊页面，点击对方消息头像时直接进入 `/character/$characterId`。
- 保持桌面端现有头像弹出卡片逻辑不变。
- 用户头像不新增跳转，避免从单聊误入“我的资料”页。

### 好友详情页
- 页面标题与返回行为保持现有路由语义，但移动端布局改成微信式单列信息页。
- 首屏信息块以“头像 / 备注名 / 昵称 / 隐界号 / 个性签名”为主，不再使用当前的图标宫格操作区。
- 中部改成微信式分组列表，优先展示：
  - `设置备注和标签`
  - `地区`
  - `来源`
  - `标签`
  - `朋友圈`
  - `更多资料`
- 关系管理区保留：
  - `设为星标朋友`
  - `加入黑名单`
  - `删除联系人`
- 底部主操作改成微信式大按钮区：
  - 好友显示 `发消息` + `音视频通话`
  - 非好友显示 `添加到通讯录`

## 实施顺序
1. 在 `chat-message-list.tsx` 增加移动端头像点击跳转逻辑，只对 `senderType === "character"` 且有 `senderId` 的消息生效。
2. 重写 `character-detail-page.tsx` 的移动端 JSX 结构，收平视觉并替换为微信式白底分组列表。
3. 复用现有 mutation 与 notice 逻辑，避免回归 `备注/标签`、`星标`、`拉黑`、`删除`、`发消息`、`朋友圈` 等现有能力。
4. 运行前端构建或类型检查验证改动可用。

## 验收口径
- 移动端打开 `/chat/$conversationId` 后，点击对方任意消息头像可进入对应 `/character/$characterId`。
- 从好友详情页返回时可回到原聊天页。
- 好友详情页移动端视觉明显收敛到微信手机端风格，不再显示当前三宫格快捷操作区。
