# Web 端电脑版群聊对标微信电脑端差异与抄写规划

## 任务目标
- 盘清当前 `apps/app` Web 端电脑版群聊工作区，和微信电脑端群聊页在功能、操作、信息架构上的真实差异。
- 只说“现在还差什么”，不重复仓库里已经做完的群聊能力。
- 输出一条按微信电脑端肌肉记忆来抄的实施顺序。

## 本次核对范围

### 本地代码基线
- 桌面消息工作区：`apps/app/src/features/desktop/chat/desktop-chat-workspace.tsx`
- 群聊线程：`apps/app/src/features/chat/group-chat-thread-panel.tsx`
- 消息列表：`apps/app/src/components/chat-message-list.tsx`
- 桌面聊天信息侧栏：`apps/app/src/features/desktop/chat/desktop-chat-details-panel.tsx`
- 桌面聊天记录侧栏：`apps/app/src/features/desktop/chat/desktop-chat-history-panel.tsx`
- 桌面会话右键菜单：`apps/app/src/features/desktop/chat/desktop-conversation-context-menu.tsx`

### 微信电脑端对标口径
- 2025-07-02，微信 `4.0.6 for Windows / Mac` 已公开新增：
  - 消息可设为提醒
  - 在聊天中可发送收藏内容
  - 独立聊天窗口和图片窗口支持小尺寸
- 2026-03-19，微信 `4.1.8` 已公开新增：
  - `Ctrl + Win` / `fn` 语音输入
  - 撤回多选转发、批量发送的图片 / 视频 / 文件
  - 视频倍速播放
- Windows 商店镜像公开信息显示，微信 `4.1.7` 已支持 `群接龙`
- 微信 Windows `3.9.6` 起已支持按`群成员`和`日期`搜索聊天记录

## 现在已经基本对齐微信电脑端的部分

### 1. 页面骨架已经不是主要矛盾
- 已经是微信电脑端熟悉的三段式：
  - 左侧会话列表
  - 中间群聊线程
  - 右侧可切换的聊天记录 / 聊天信息侧栏
- 顶部群聊头部已经收敛为：
  - 群名
  - 成员数
  - 右上角按钮

### 2. 群聊消息区基础操作已经不算缺
- 已有：
  - 回复
  - 部分引用
  - 转发
  - 多选
  - 提醒
  - 收藏
  - 撤回
  - 删除
  - 图片 / 文件打开与另存
- 群聊输入区已有：
  - `@` 成员候选
  - 表情
  - 图片 / 文件 / 名片 / 位置发送
  - 语音转文字入口
  - 粘贴文件
  - 拖拽文件

### 3. 群聊设置同步链路已经接上
- 群聊信息侧栏已经能直接操作并同步：
  - 免打扰
  - 置顶聊天
  - 保存到通讯录
  - 我在本群的昵称
  - 显示群成员昵称
  - 聊天背景
  - 清空记录
  - 隐藏聊天
  - 删除并退出

### 4. 聊天记录侧栏也不是从零开始
- 已支持：
  - 关键词搜索
  - 类型筛选
  - 日期筛选
  - 群成员筛选
  - 点击结果回跳并高亮消息

## 还和微信电脑端有明显差异的地方

## P0：先补这些，用户才会觉得“真的像微信电脑版群聊”

### 1. 通话入口是假能力，不是微信那种可直接用的群通话
- 现在顶部通话按钮只是把用户导去 `桌面/手机接力` 路由，并没有真正群语音 / 群视频能力。
- 微信电脑端当前口径已经是：
  - 群聊视频通话
  - 桌面端语音输入
  - 更完整的电脑端通话工作流
- 结论：
  - 这个是当前“最像占位”的地方，优先级必须最高。

### 2. 缺 `群接龙`
- 微信 Windows `4.1.7` 已支持 `群接龙`。
- 当前群聊没有任何接龙入口、接龙消息类型、接龙编辑器，也没有群侧栏入口。
- 这会直接导致“办公型微信群”的关键习惯动作缺位。

### 3. 缺视频消息能力，导致和微信电脑端最新消息操作不在一个量级
- 当前群聊附件只到：
  - 图片
  - 文件
  - 名片
  - 位置
  - 表情
- 微信电脑端公开口径已经覆盖：
  - 图片 / 视频 / 文件批量发送
  - 视频倍速播放
  - 撤回批量发送的图片 / 视频 / 文件
- 当前缺口不只是“少一个附件类型”，而是整条视频消息链路都缺：
  - 上传
  - 渲染
  - 打开
  - 倍速播放
  - 转发
  - 撤回
  - 搜索归类

### 4. 收藏内容不能直接发回群聊
- 项目里已经有“收藏”工作区，也有“消息收藏”。
- 但微信群聊输入区没有“发送收藏内容”入口。
- 微信 `4.0.6` 已公开支持“在聊天中可发送收藏内容”。
- 这意味着我们虽然有收藏数据层，但还没做成微信电脑端那种闭环。

### 5. 聊天记录检索仍然只是“最近消息里的前端过滤”，不是微信那种完整检索
- 当前右侧聊天记录侧栏虽然有筛选，但底层还是：
  - 先拉最近 `80` 条
  - 逐步继续加载
  - 前端本地过滤
  - 最多只展示 `60` 条结果
- 微信电脑端至少已经明确支持：
  - 按群成员搜索
  - 按日期搜索
- 当前实现的问题不是“没有筛选 UI”，而是：
  - 数据覆盖面不完整
  - 没有日历式时间跳转
  - 长群聊历史会漏结果
  - 结果列表不是微信那种“真正查全量索引”的心智

### 6. 群成员区域还太浅，不像微信电脑端的完整群资料面板
- 当前右侧群成员宫格只展示前 `8` 个成员，再补 `添加 / 移除` 按钮。
- 还缺微信电脑端常见的这些动作：
  - 查看完整成员列表
  - 搜索成员
  - 点成员头像进资料
  - 更明确地展示群主 / 管理员 / 普通成员
  - 更像微信的“更多群成员”展开方式
- 这块现在更像“能改数据”，不像“能管理群”。

## P1：会影响完成度，但可以排在主链路之后

### 7. 缺独立聊天窗口
- 当前只有图片能独立窗口打开。
- 微信 `4.0.6` 公开口径里已经包含“独立聊天窗口和图片窗口支持小尺寸”。
- 如果要完全抄微信电脑端，群聊会话本身也要能弹出独立窗口，而不是只能留在主工作区里。

### 8. 群公告的电脑端表达还不够像微信
- 当前桌面端群公告主要停留在右侧详情面板中的一行入口和一个编辑弹窗。
- 缺少更微信化的几个点：
  - 更强的公告摘要表达
  - 更明确的公告发布者 / 更新时间
  - 公告富内容承载预留
  - 公告和群接龙、群文件这类“群协作能力”的并列关系
- 如果后面要继续抄微信办公群场景，群公告需要升级成一个明确的群协作模块，不只是纯文本字段。

### 9. 右键菜单虽然能用，但“排法”和“消息类型覆盖”还没完全按微信桌面端来
- 现在功能已经不少，但还不够微信式：
  - 菜单命令排序还是项目自己的顺序
  - 缺视频消息后，视频相关命令整体缺席
  - 缺从收藏发送后，聊天与收藏工作流没有连上
  - 没有把群接龙这类群消息对象纳入菜单体系
- 这块不需要大改底层，更多是等 P0 能力补齐后统一重排。

### 10. 视觉气质还不是微信电脑端
- 当前群聊线程和侧栏仍然用了偏暖的背景渐变、圆角卡片和投影。
- 微信电脑端更接近：
  - 冷白灰
  - 更扁平
  - 更弱装饰
  - 更像系统桌面工具
- 如果目标是“完全抄微信”，视觉 token 需要再收一次，不然功能补齐后第一眼还是不像。

## 建议的抄写顺序

## Phase 1：先把“微信当前已公开的新能力”补上
- 目标：先消灭最明显的代差。
- 要做：
  - 真正群语音 / 群视频能力落地
  - 群接龙消息与入口
  - 视频消息全链路
  - 从收藏发送到群聊
- 主要改动面：
  - `apps/app/src/components/chat-composer.tsx`
  - `apps/app/src/components/chat-message-list.tsx`
  - `apps/app/src/features/chat/group-chat-thread-panel.tsx`
  - `apps/app/src/features/desktop/chat/desktop-chat-header-actions.tsx`
  - `packages/contracts`
  - `api/src/modules/chat`

## Phase 2：把右侧群资料和聊天记录做成真正的微信桌面群工具区
- 目标：补齐“管理群”和“找历史”这两条微信高频流。
- 要做：
  - 群成员完整列表 / 搜索 / 资料跳转
  - 群主 / 管理员表达
  - 群公告模块升级
  - 服务端聊天记录检索
  - 日历跳转和更完整结果集
- 主要改动面：
  - `apps/app/src/features/desktop/chat/desktop-chat-details-panel.tsx`
  - `apps/app/src/features/desktop/chat/desktop-chat-history-panel.tsx`
  - `api/src/modules/chat/group.service.ts`
  - `packages/contracts/src/chat.ts`

## Phase 3：最后再做“微信电脑端效率味道”
- 目标：把“像”推进到办公效率层。
- 要做：
  - 群聊独立窗口
  - 更克制的微信化视觉 token
  - 右键菜单顺序和按类型裁剪全面对齐
- 主要改动面：
  - `apps/app/src/features/desktop/chat`
  - `apps/app/src/features/shell`
  - `apps/app/src/components/chat-message-list.tsx`

## 一句话结论
- 这轮最真实的判断不是“我们还差一堆基础群聊功能”，而是：
  - 基础群聊已经有了
  - 现在卡住微信电脑端完成度的，是 `群通话 / 群接龙 / 视频消息 / 收藏回发 / 全量历史检索 / 完整群成员管理`
- 先把这六块补齐，再统一压视觉和菜单顺序，桌面群聊页就会非常接近微信电脑端。
