# 主 App 移动端 Tab 固定顶部栏规划

## Summary
- 在 `apps/app` 的移动端 `tabs` 页面补一层统一的顶部固定栏，行为对齐微信：除“我”页外，其余 tab 进入页面后顶部始终显示当前所在 tab 名称，并在滚动时保持吸顶。
- 重点先解决消息页空态时缺少页头、页面显得发空的问题；桌面端现有大页头不动。

## Key Changes
- 新增一个移动端专用的通用顶部栏组件，输入最少包含 `title`，可选 `rightActions`，内置 `sticky top-0`、半透明背景、底部分隔线、`backdrop-blur`。
- 在移动端 4 个 tab 页接入该组件：
  - `消息`：始终显示“消息”，不再依赖 `hasConversations` 决定是否渲染页头。
  - `朋友圈`：显示“朋友圈”。
  - `发现`：显示“发现”。
  - `通讯录`：显示“通讯录”。
- `我` 页保持现状，不加这层 tab 顶部固定栏，避免和个人资料大卡头重复。
- `AppHeader` 继续承担页面说明卡片角色，不改成全局 sticky。
- 消息页空态补足明确内容区和引导动作，不再只剩空白滚动区。

## Public Interfaces / Types
- 新增一个前端内部复用组件接口：
  - `title: string`
  - `rightActions?: ReactNode`
  - `className?: string`
- 不新增后端 API。
- 不修改现有业务 contracts。

## Test Plan
- 验证 `/tabs/chat`、`/tabs/moments`、`/tabs/discover`、`/tabs/contacts` 首屏立即能看到顶部固定栏。
- 验证滚动时顶部栏持续吸顶，不与底部 tab bar 发生层级冲突。
- 验证消息页有会话和无会话两种状态下顶部栏都存在。
- 验证“我”页不新增固定 tab 顶栏。
- 验证桌面端布局不变。

## Assumptions
- 本次范围只处理移动端 Tab，不处理桌面端统一顶栏。
- 固定栏使用“仅标题 + 轻操作”。
- “我”页继续排除在统一 tab 顶栏之外。
- 文案统一使用中文 tab 名称。
