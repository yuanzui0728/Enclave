# Web端电脑版 UI/UX 整体优化规划

日期：2026-04-11
仓库：`/home/ps/claude/yinjie-app`
范围：`apps/app` 的 Web 端电脑版工作区

## 任务定义

- 本轮目标不是零散改几个页面，而是把 Web 端电脑版做一次整体 UI/UX 收口。
- UX 目标明确按微信电脑端走：
  - 左侧主导航稳定
  - 二级列表低噪音
  - 主工作区内容优先
  - 右侧详情按需展开
  - 搜索、资料、文件、工具都围绕“工作区”而不是“专题页”组织
- UI 目标不是完全照搬微信视觉，而是在微信电脑端交互骨架上，做一套“积极向上但不过度兴奋”的桌面风格：
  - 更明亮
  - 更清爽
  - 更有生机
  - 但不能抢走信息层级
- 这份规划只覆盖 Web 端电脑版，不扩散到：
  - 移动端主路径
  - Tauri 原生壳专属行为
  - 后端接口语义调整

### 当前实现的真实状态

- 桌面端已经有完整的壳层和大量独立工作区，不是从零开始。
- 消息页、通讯录页已经接近微信电脑端的三栏工作区模式。
- 搜索页也已经是“左分类 + 中结果 + 右说明”的桌面结构。
- 但收藏、反馈、聊天记录、设置、占位页等大量页面仍然建立在 `DesktopEntryShell` 之上，呈现为：
  - 左侧大标题
  - 右侧卡片区
  - 大段说明文案
  - 偏 landing page 的信息排布
- 这类页面在视觉上更“会展示”，但在体验上不像微信电脑端的工具页。

### 当前主要问题

#### 1. 视觉方向和 UX 方向不一致

- `packages/ui/src/tokens.css` 目前整体是暖橙、奶油、琥珀阴影。
- `apps/app/src/features/shell/desktop-shell.tsx` 又继续叠加了：
  - 橙黄发光背景
  - 大圆角壳
  - 强装饰标题栏
  - `YJ / DESKTOP WORLD` 品牌块
- 结果是：
  - UI 很积极
  - 但不够克制
  - 和微信 PC 的稳定效率感冲突

#### 2. 核心页面与工具页面没有统一桌面语义

- `desktop-chat-workspace.tsx`、`contacts-page.tsx` 的结构更接近微信电脑端。
- `favorites-page.tsx`、`desktop-feedback-page.tsx`、`desktop-chat-history-page.tsx`、`profile-settings-page.tsx` 仍然偏“专题页 / 看板页”。
- 用户在同一个桌面端里切换频道时，会感觉：
  - 有的页面像 IM
  - 有的页面像官网后台
  - 有的页面像概念展示页

#### 3. 文案语气偏产品展示，不像即时通讯桌面客户端

- 例如：
  - “收藏把跨频道的重要内容统一收住”
  - “意见反馈把问题和上下文一起交付”
  - “桌面端按微信式工作区组织搜索范围、结果列表和最近记录”
- 这些文案放在规划稿里没问题，放在真实桌面产品里会显得过满。
- 微信电脑端的文案通常更短、更功能化、更弱存在感。

#### 4. 过度使用大卡片、大圆角、大阴影

- 当前很多桌面页默认：
  - `18px+` 圆角
  - 成组阴影卡片
  - 大面积留白
  - 多块统计卡
- 这会带来两个问题：
  - 信息密度低
  - 页面更像内容运营看板，不像长期高频使用的桌面工具

#### 5. 搜索语义仍有偏差

- 消息页搜索框在 `desktop-chat-workspace.tsx` 里还是“列表过滤 + 跳搜一搜”混合语义。
- 微信电脑端更接近：
  - 左上搜索框首先是全局搜索入口
  - 列表过滤是弱能力
- 现有实现不是不能用，但还不够“像微信”。

## 总体设计原则

## 一句话策略

- UX 抄微信电脑端
- UI 做隐界自己的积极版本
- 但积极感来自“清爽、自然、向上”，不是“高饱和、大氛围、大营销”

### 1. 微信式桌面交互不变量

- 左侧主导航永远稳定，不抢注意力。
- 二级列表承担选择，不承担解释。
- 解释性交互尽量进入右侧面板，而不是堆在主界面首屏。
- 主工作区优先服务“当前任务”，不是先讲故事。
- 能直接进入工作态的页面，不再做 Hero 区。
- 搜索、资料、工具页都要有明确的“选择对象 -> 查看详情 -> 执行动作”闭环。

### 2. 积极向上视觉不变量

- 主色从当前橙色系改成“微信绿为锚点，浅日光绿为辅”的轻亮体系。
- 页面基底以白、雾灰、极浅暖灰为主，彩色只出现在：
  - 激活态
  - 主按钮
  - 状态提示
  - 小面积强调
- 背景允许保留一点“晨光感”，但必须降到第二层，不能形成大面积发光舞台。
- 阴影只表达层级，不表达情绪。
- 圆角收紧，统一成微信 PC 近似密度。

### 3. 桌面端组件语义不变量

- 不再把“统计卡 + 说明卡 + 标语标题”作为桌面页默认模板。
- 后续桌面页统一分成三类壳：
  - `DesktopWorkspaceShell`
    - 用于消息、通讯录、搜索、聊天记录、收藏这类高频工作区
  - `DesktopUtilityShell`
    - 用于设置、反馈、手机联动、直播伴侣这类工具页
  - `DesktopImmersiveShell`
    - 用于独立窗口、图片查看器、通话页这类沉浸场景

## 目标视觉方向

### 色彩

- 主品牌高频色：`#07C160` 附近的绿色体系
- 桌面辅助色：
  - 清透浅绿
  - 中性雾灰
  - 极浅暖白
- 删除当前高频使用的：
  - 大面积橙色发光
  - 黄橙渐变品牌块
  - 暖棕文字体系

### 排版

- 标题层级收紧：
  - 一级页标题不再动辄 32px 到 42px
  - 高频工具页改为 16px 到 20px 的桌面标题
- 辅助说明控制在一行到两行内。
- 减少全大写英文、副标题口号和追求“品牌感”的字距。

### 圆角与阴影

- 导航按钮：8px 到 10px
- 列表项：8px 到 10px
- 面板：10px 到 12px
- 弹层：12px 到 16px
- 大壳圆角整体收紧，不再默认 `30px+`
- 阴影统一用弱层级阴影，去掉当前偏暖且存在感强的投影

### 动效

- 保留进入动效，但只用于：
  - 页面切换
  - 抽屉展开
  - 菜单显隐
  - hover 激活
- 移除“为了氛围”的大块发光和漂浮感。
- 动效曲线对齐微信式快响应，不做明显弹跳。

## 架构级改造方向

### A. 桌面壳层重构

涉及文件：

- `packages/ui/src/tokens.css`
- `apps/app/src/index.css`
- `apps/app/src/features/shell/desktop-shell.tsx`
- `apps/app/src/features/shell/desktop-nav-config.ts`

改造目标：

- 让桌面壳从“品牌化外壳”收口为“低噪音工作容器”。

具体动作：

- 把 `tokens.css` 的桌面默认色系从暖橙琥珀收紧到轻绿中性体系。
- `body` 背景保留极浅环境层，但取消强烈橙黄光斑。
- `desktop-shell.tsx` 的顶部品牌块改成更克制的应用头：
  - 左侧应用名
  - 当前世界主人信息
  - 可选状态点
- 左侧主导航改成更像微信 PC 的：
  - 深一点的竖向导航带
  - 激活态明确
  - 非激活态低存在感
- “更多”菜单改成标准桌面菜单，不再做带说明卡的大浮层。

验收标准：

- 不看内容也能感觉更像桌面 IM，而不是桌面官网。
- 主导航和内容区的主次关系更明确。

### B. 统一桌面工作区模板

涉及文件：

- `apps/app/src/features/desktop/desktop-entry-shell.tsx`
- `apps/app/src/features/desktop/desktop-placeholder-workspace.tsx`
- `apps/app/src/routes/favorites-page.tsx`
- `apps/app/src/routes/desktop-feedback-page.tsx`
- `apps/app/src/routes/desktop-chat-history-page.tsx`
- `apps/app/src/routes/profile-settings-page.tsx`
- `apps/app/src/routes/desktop-mobile-page.tsx`
- `apps/app/src/routes/live-companion-page.tsx`

问题判断：

- `DesktopEntryShell` 当前更像宣传型双栏布局，不适合作为桌面工具页母版。

改造策略：

- 不继续在 `DesktopEntryShell` 上叠补丁。
- 新建更克制的工具页壳，原则是：
  - 顶部是工具标题和操作
  - 中部是主要工作区域
  - 说明信息退到右侧或底部
  - 数据摘要只在必要时出现

预期结果：

- 收藏页变成真正的收藏管理器，不再像“收藏介绍页”。
- 反馈页变成问题提交与历史记录工具，不再像“反馈理念页”。
- 聊天记录页变成“会话列表 + 记录视图 + 操作栏”的桌面检索页。
- 设置页更接近微信 PC 设置结构，而不是单纯把移动设置页放大。

### C. 消息工作区精修为桌面基准页

涉及文件：

- `apps/app/src/features/desktop/chat/desktop-chat-workspace.tsx`
- `apps/app/src/features/desktop/chat/desktop-chat-side-panel.tsx`
- `apps/app/src/features/desktop/chat/desktop-chat-header-actions.tsx`
- `apps/app/src/features/chat/conversation-thread-panel.tsx`
- `apps/app/src/features/chat/group-chat-thread-panel.tsx`

当前判断：

- 这是全仓最接近微信 PC 的页面，应该作为桌面视觉和交互母版。

优化重点：

- 左栏搜索框语义收口：
  - 默认视为“全局搜索入口”
  - 本地过滤降级为次级能力
- 左栏会话项密度、字重、badge、未读、免打扰图标全部统一
- 快捷菜单改成更像微信的轻菜单
- 顶栏按钮更扁平、更轻、更贴边
- 右侧详情面板的头部和分组更像微信，而不是“后台设置块”
- 会话空态不能再是纯空白，至少给出微信式轻提示

验收标准：

- 消息页成为所有桌面页的视觉基准页。
- 后续桌面模块都向它靠，而不是各做各的。

### D. 通讯录页收口为“微信通讯录电脑版”风格

涉及文件：

- `apps/app/src/routes/contacts-page.tsx`
- `apps/app/src/features/contacts/contact-detail-pane.tsx`
- `apps/app/src/features/contacts/desktop-contact-profile-blocks.tsx`

当前判断：

- 通讯录结构是对的，但视觉还偏暖、偏卡片化。

优化重点：

- 左栏 shortcut 卡片改弱，减少彩色 icon 大块
- 联系人列表更接近微信 PC：
  - 字高更稳
  - 行间距更统一
  - 分组标题更轻
- 右侧详情改成“资料页”而不是“资料运营面板”
- 联系人操作按钮收口到：
  - 发消息
  - 音视频
  - 备注 / 标签 / 置顶 / 免打扰
- 删除强营销式留白和大卡块

### E. 搜索页从“搜索专题页”收口成“桌面搜一搜工作台”

涉及文件：

- `apps/app/src/features/search/desktop-search-workspace.tsx`
- `apps/app/src/routes/search-page.tsx`

当前判断：

- 搜索页的三栏结构可用，但首屏解释过多。

优化重点：

- 左栏改成更接近微信分类栏
- 中栏保留核心搜索框和结果列表
- 右栏从“说明文案”切成：
  - 搜索提示
  - 最近搜索
  - 当前命中摘要
  - 结果预览
- 未输入关键词时，不再堆三张说明卡，改为微信式搜索建议和历史

### F. 收藏、聊天记录、设置、反馈等工具页全部微信化

涉及文件：

- `apps/app/src/routes/favorites-page.tsx`
- `apps/app/src/routes/desktop-chat-history-page.tsx`
- `apps/app/src/routes/profile-settings-page.tsx`
- `apps/app/src/routes/desktop-feedback-page.tsx`

改造原则：

- 不再出现“工具页开头先讲价值主张”的写法。
- 页面必须在首屏直接进入操作态。

对应目标：

- 收藏页：
  - 左侧分类
  - 中间列表
  - 右侧详情/预览
- 聊天记录页：
  - 左侧会话
  - 中间记录列表
  - 顶部筛选和操作
- 设置页：
  - 左侧设置分类
  - 中间表单
  - 去掉移动端式顶栏回退语义
- 反馈页：
  - 左侧反馈分类与历史
  - 中间表单
  - 右侧诊断摘要

### G. 内容流页面收口到统一桌面阅读体验

涉及文件：

- `apps/app/src/features/desktop/moments/*`
- `apps/app/src/features/desktop/feed/*`
- `apps/app/src/features/desktop/channels/*`
- `apps/app/src/routes/moments-page.tsx`
- `apps/app/src/routes/feed-page.tsx`
- `apps/app/src/routes/channels-page.tsx`

当前判断：

- 内容流页面已经形成自己的结构，但和消息/通讯录的桌面语言仍不统一。

优化重点：

- 工具条高度、边框、按钮样式与消息页统一
- 左右面板分隔、背景层级统一
- 内容卡片去掉过强的“设计稿感”
- 让朋友圈、广场、视频号看起来像同一客户端里的不同频道，而不是三个不同产品

- 空态、错误态、加载态
- 右侧面板开合动画
- 独立页返回链路视觉一致性

提交建议：

- `feat(app): finish desktop ux polish pass`

## 需要统一的设计规范

### 组件级

- 按钮只保留三种主语义：
  - 主按钮
  - 次按钮
  - 幽灵按钮
- 搜索框统一高度和图标位置
- 列表项统一：
  - 行高
  - 头像尺寸
  - 标题字号
  - 副文案颜色
  - 激活背景
- 右侧详情面板统一：
  - 标题头
  - section 间距
  - 分割线
  - 行式设置项

### 页面级

- 一级桌面页禁止默认使用 Hero 标题区
- 一级桌面页禁止默认使用三张以上统计卡
- 一级桌面页首屏必须有真实可操作内容
- 文案超过两行的解释默认后移，不占首屏

## 非目标

- 本轮不追求把每个模块都做成完全 1:1 微信复刻视觉稿。
- 本轮不新增大后端能力，只优化已有体验和结构。
- 本轮不优先处理移动端视觉统一。

## 风险与约束

### 1. 现有页面样式耦合较深

- 目前很多页面直接写死：
  - `bg-[#f3f3f3]`
  - `border-black/6`
  - 大圆角
  - 局部自定义阴影
- 这意味着不能只改 token，必须同步改页面级 class。

### 2. `DesktopEntryShell` 使用面过广

- 不能一次性全替换，否则回归面太大。
- 更稳妥的做法是：
  - 先引入新母版
  - 再逐页迁移

### 3. 当前仓库已有未提交变更

- 目前工作区里已有用户自己的修改：`apps/app/src/routes/desktop-chat-image-viewer-page.tsx`
- 本轮执行时不能误带入或覆盖该文件。

## 完成定义

满足以下条件才算这轮桌面 UI/UX 优化完成：

1. 桌面端所有一级入口共享同一套壳层和设计语言。
2. 消息、通讯录、搜索、收藏、设置、反馈、聊天记录不再有明显“专题页”感。
3. UI 观感更积极、更清爽，但不会压过信息内容。
4. UX 明显向微信电脑端靠拢，尤其是：
   - 导航
   - 列表
   - 主工作区
   - 右侧详情
   - 搜索
5. 所有改动按小步提交推进，不混入用户已有改动。

## 执行建议

- 真正开工时，不要按页面随意挑着改。
- 应严格按照这个顺序推进：

1. 先改 token 和桌面壳
2. 再建立新的桌面母版
3. 再精修消息页和通讯录页
4. 再迁移工具页
5. 最后统一内容流与细节

- 这样能保证每一轮改造后，整个桌面端都更一致，而不是出现新旧两套设计继续并存。
