# Web端电脑版消息页对标微信聊天页总规划

## 任务目标
- 目标范围只覆盖 `apps/app` 的 Web 端电脑版消息工作区，不动移动端主路径。
- 目标不是“做一个像微信的聊天壳”，而是把消息列表、聊天线程、右侧信息、输入区、消息操作、搜索与文件流转一起收敛到当前微信电脑端的使用习惯。
- 这份规划作为桌面消息页母计划，统一收口以下几个已拆开的专项：
  - `web端电脑版聊天页右上角按钮与聊天信息微信化-2026-04-10.md`
  - `web端电脑版群聊功能微信化规划-2026-04-10.md`
  - `web端聊天页表情包功能规划-2026-04-09.md`
  - `web端聊天页语言输入功能-2026-04-09.md`

  - `2025-11-28`：文件直接下载到指定目录、滚动截图

### 需要明确标注的推断
- 官方版本历史不会把所有交互细节逐条列全。
- 因此以下内容属于高可信推断，并按“当前微信电脑端长期稳定交互”纳入规划：
  - 会话列表右键菜单
  - 消息右键菜单
  - 图片查看器独立打开
  - 更扁平的灰白三栏桌面工作区

## 当前桌面消息页已经有的能力
- 左侧三栏消息工作区、会话搜索、置顶态、未读 badge、免打扰图标
- 单聊 / 群聊收发
- 图片 / 文件 / 名片 / 位置卡片 / 表情包
- 群聊 `@` 候选、群消息整条引用回复
- 右上角 `聊天记录 / 通话 / 更多`
- 右侧 `聊天记录 / 聊天信息` 双模式侧栏
- 聊天背景、会话置顶、清空记录、拉黑、投诉、群成员管理、群公告修改等部分能力
- 桌面独立页：
  - 聊天记录管理
  - 聊天文件
  - 笔记

## 差异总览

### A. 左侧消息列表还不像微信

#### 1. 搜索语义不对
- 当前左上搜索框只是本地过滤 `会话标题 + 最后一条预览`。
- 微信电脑端消息页左上搜索更接近“全局搜索入口”，至少会跨：
  - 联系人
  - 群聊
  - 聊天记录
  - 文件 / 文章 / 收藏等
- 结论：
  - 现在的搜索框更像列表筛选器，不像微信的消息页搜索入口。

#### 2. 会话项操作不够
- 当前会话卡片只有点击进入，没有桌面 IM 该有的操作闭环。
- 缺口：
  - 会话右键菜单
  - 标为未读
  - 取消 / 设置置顶的快捷操作
  - 免打扰快捷操作
  - 隐藏聊天 / 删除聊天的就地操作
- 结论：
  - 左栏目前“能看”，但“不够用”。

#### 3. 会话项表现还不完整
- 当前群聊卡片已支持 `发送者前缀 + 内容`，但还缺更细的微信式状态表达：
  - `@我`
  - `@所有人`
  - 草稿
  - 撤回提示
  - 群公告 / 置顶消息带来的会话摘要变化
- 结论：
  - 左栏信息密度和微信还有明显差距。

### B. 中间聊天线程只完成了骨架

#### 1. 单聊缺消息级操作
- 当前单聊桌面线程没有右键菜单。
- 群聊桌面线程虽然有右键菜单，但只有：
  - 回复
  - 复制消息
  - 复制发送者
- 与微信差异：
  - 没有转发
  - 没有多选
  - 没有撤回
  - 没有收藏
  - 没有删除
  - 没有引用部分文字
  - 没有设为提醒
- 结论：
  - 当前消息区只能“看和发”，不能像微信那样高频处理消息。

#### 2. 单聊和群聊能力不对称
- 群聊已支持整条引用回复。
- 单聊仍没有回复链路。
- 这和微信电脑端实际使用习惯不一致。
- 结论：
  - 需要把引用 / 回复模型收敛成单聊群聊共用能力。

#### 3. 消息类型还不够全
- 当前没有视频消息气泡与视频查看流。
- 微信桌面端当前已经把视频相关体验继续往前推，包括：
  - 视频引用回复
  - 视频倍速播放
- 结论：
  - 如果要“完全抄微信聊天页”，视频消息必须补。

#### 4. 群置顶消息 / 群公告主线程承接不够
- 当前桌面群聊主线程没有承接群置顶消息。
- 群公告也没有形成微信式的桌面主线程顶部承载。
- 结论：
  - 群聊线程还缺“运营层信息”这一层。

### C. 输入区与发送流程差距最大

#### 1. 输入器结构还不是微信桌面输入器
- 当前桌面输入框是单行 `input`，不是桌面聊天常见的可扩展输入区域。
- 已有：
  - 表情
  - 图片 / 文件
  - 语音转文字
  - 粘贴图片 / 文件
- 缺口：
  - 截图入口
  - 滚动截图
  - 截图打码
  - 截图置顶桌面
  - 拖拽文件到聊天区发送
  - 发送收藏内容
  - 视频发送
  - 表情搜索
- 结论：
  - 现在更像“网页版 IM 输入条”，不像微信桌面工具条。

#### 2. 语音输入只做了浏览器版，不是微信桌面版
- 当前实现是 Web 语音转文字入口。
- 微信 `4.1.8` 公开信息已经把语音输入做成了桌面级快捷操作。
- 结论：
  - Web 端可以保留现有实现，但交互上至少要对齐：
    - 按钮位置
    - 状态文案
    - 持续输入体验

#### 3. 回复和发送的组合能力不够
- 当前群聊回复只支持“整条消息 + 文本 / 图片 / 文件 / 名片 / 位置 / 表情”中的部分。
- 微信近版已经支持：
  - 引用部分文字
  - 引用时回复图片 / 视频 / 表情
- 结论：
  - 回复模型不能再只靠把元数据塞进文本体。
  - 需要显式消息引用结构。

### D. 右侧栏是半成品

#### 1. 单聊信息侧栏还在用本地偏好兜底
- `消息免打扰` 现在在桌面单聊详情里还是本地 `localStorage` 偏好。
- 后端其实已有 `setConversationMuted` 契约，但桌面侧栏没接。
- 同类问题还包括：
  - `强提醒` 是项目自定义偏好，不是微信电脑端常用桌面表述
  - 没有 `隐藏聊天`
- 结论：
  - 右侧单聊信息区必须改成真数据，不要再混本地伪设置。

#### 2. 群聊信息侧栏缺真实设置项
- 移动端群详情已接了这些能力，但桌面右侧栏没有同步：
  - `@我仍通知`
  - `@所有人仍通知`
  - `群公告仍通知`
  - `保存到通讯录`
  - `隐藏聊天`
  - `删除并退出`
- 当前桌面群右栏还在用本地偏好存：
  - `消息免打扰`
  - `显示群成员昵称`
- 结论：
  - 桌面侧栏的完整度落后于移动端详情页。

#### 3. 聊天记录侧栏只是前端过滤器
- 当前右侧 `聊天记录` 面板只有：
  - 全量拉消息
  - 前端字符串过滤
  - 最多展示最近 `60` 条结果
- 与微信差异：
  - 不能按群成员筛
  - 不能按日期筛
  - 不能按消息类型筛
  - 没有更长历史搜索
  - 没有附件聚合搜索
- 结论：
  - 现在像“临时搜索抽屉”，不像微信桌面聊天记录检索。

### E. 图片、文件、视频工作流不完整

#### 1. 图片查看器缺失
- 当前图片消息只是直接渲染 `<img>`。
- 缺口：
  - 大图查看层
  - 多图切换
  - 定位到聊天位置
  - 独立图片窗口
  - 打印图片
- 结论：
  - 这是最明显的“和微信不像”的点之一。

#### 2. 聊天文件页还是过渡态
- 当前 `desktop-chat-files-page` 还是按单会话聚合，且页面自己也写明“后续补跨会话筛选与搜索”。
- 与微信差异：
  - 不能像微信那样把聊天文件当成完整桌面工作流
  - 没有指定下载目录
  - 没有未下载文件直接保存
  - 没有从附件回跳原消息
- 结论：
  - 现在只是附件列表，不是微信式聊天文件能力。

#### 3. 视频消息和视频播放器缺位
- 当前没有视频消息模型，也没有视频播放器。
- 微信桌面端当前已经把视频播放体验继续加强。
- 结论：
  - 视频必须补到消息主链路，不适合继续放在后面。

### F. 独立窗口工作流完全没做
- 微信桌面端已经长期支持：
  - 独立聊天窗口
  - 图片窗口
  - 小尺寸窗口
- 当前 Web 端没有：
  - 独立会话窗口
  - 独立图片查看窗口
  - 浮动工作区
- 结论：
  - 如果用户经常一边聊一边处理别的内容，这块差异非常大。

### G. 视觉与操作节奏还不是微信
- 当前桌面消息页仍然大量使用：
  - 米色渐变
  - 大圆角卡片
  - 暖色阴影
- 微信电脑端近版明显更平、更灰白、更工具化。
- 结论：
  - 如果只补功能不改表面层，用户还是会觉得“不像微信”。

## 差异优先级

- 左侧搜索从“会话过滤”升级为“消息页全局搜索入口”
- 会话列表右键菜单与常用快捷操作
- 单聊 / 群聊统一消息右键菜单
- 单聊补引用回复
- 图片查看器
- 视频消息与视频播放器

  - 拖拽发送
  - 表情搜索
  - 视频发送

### 前端必须重做的部分
- `apps/app/src/features/desktop/chat/desktop-chat-workspace.tsx`
- `apps/app/src/features/chat/conversation-thread-panel.tsx`
- `apps/app/src/features/chat/group-chat-thread-panel.tsx`

- `apps/app/src/features/desktop/chat/desktop-chat-history-panel.tsx`
- `apps/app/src/routes/desktop-chat-files-page.tsx`

### 后端 / 契约必须补的部分
- 消息引用结构从“文本编码”升级为显式字段
- 消息操作接口：
  - 撤回
  - 删除
  - 转发辅助能力
  - 多选批量动作
- 聊天记录搜索接口：
  - 关键词
  - 群成员
  - 日期
  - 消息类型
- 视频消息契约
- 文件下载偏好与下载目录配置

## 不建议混进这一轮的内容
- 收付款 / 转账完整能力
- 真实 RTC 音视频通话
- 多用户在线态 / 已读回执体系
- 和手机端完全一致的扫码登录链路

原因：
- 这些不是“桌面消息页像不像微信”的首要阻塞。
- 有些还会直接拉出支付、设备、账号体系的额外工程量。

## 结论
- 当前桌面消息页已经有了三栏聊天工作区的基础，但距离“微信电脑端聊天页”还差三大块：
  - 消息操作闭环
  - 媒体 / 文件 / 搜索工作流
  - 更平更硬朗的桌面表层
- 如果按“完全抄微信”的优先级来排，最应该先做的不是再加零散入口，而是一次性把：
  - 左栏搜索与会话操作
  - 中间消息操作
  - 输入器
  - 右栏真设置
  - 图片 / 视频查看器
 这五块一起收拢。
