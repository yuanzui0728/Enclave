# Web 端电脑版群聊对齐微信电脑端二次规划

## 任务目标
- 重新核对当前 `apps/app` 的桌面群聊工作区，明确它和微信电脑端群聊页在功能、入口、操作顺序上的真实差异。
- 不沿用昨天已经过时的判断，按 2026-04-11 当前代码基线重新列缺口。
- 输出一份“能直接照着抄微信电脑版群聊”的实施顺序。

## 微信电脑端参考口径

### 已核对的微信 PC 端公开信息
- 2023-06-20，微信 Windows `3.9.6` 已支持按群成员和日期搜索聊天记录。
- 2025-01-22，微信 PC `4.0.2` 内测已支持群视频通话共享屏幕、截图一键打码、聊天记录双向迁移、存储空间管理。
- 2025-08-06，微信 Windows `4.1.0` 内测已支持部分引用、引用回复图片/视频/表情、同时打开多张图片、截图置顶桌面、主窗口置顶、拖拽拉出独立聊天窗口。
- 2026-03-05，微信 Windows / Mac `4.1.8.21` 已把聊天输入区功能入口收口到底部，公开展示了“表情 / 发送收藏 / 发送文件 / 截图 / 聊天记录 / 语音输入文字”这一套桌面输入条心智。
- 2026-03-19，微信 `4.1.8` 已支持 `Ctrl + Win` / `fn` 语音输入、撤回多选转发和批量发送的图片/视频/文件、视频倍速播放。

## 结论先说
- 当前项目的桌面群聊已经不是“功能很少”，而是“很多能力有了，但微信式入口和微信式操作流还没抄到位”。
- 真正差的不是群聊 CRUD，而是四件事：
  - 通话和协作能力还是假的或不完整
  - 输入区与右键菜单的微信肌肉记忆不对
  - 检索、收藏、提醒仍有明显“本地玩具化”痕迹
  - 视觉和布局仍偏项目自定义，不像微信桌面工具

## 和微信电脑端仍有明显差异的地方

## P0：必须先抄到位

### 1. 群通话仍是假入口
- 现在桌面群聊顶部“通话”按钮并不会直接发起可用的群语音 / 群视频，只是跳到 `/desktop/mobile` 接力页。
- 微信电脑端至少从 2020 年开始就已有群语音 / 群视频通话，2025-01-22 的 `4.0.2` 公开信息里又明确出现了群视频共享屏幕。
- 这会导致桌面群聊最像微信的一个高频动作，在我们这里仍是占位符。

### 2. 还缺视频消息全链路
- 当前群聊消息类型只有文本、表情、图片、文件、语音、名片、位置。
- 微信 `4.1.8` 已明确覆盖批量发送图片 / 视频 / 文件，以及视频倍速播放。
- 如果不补视频消息，下面一整串都会持续和微信错位：
  - 输入区入口
  - 消息气泡
  - 预览器
  - 转发
  - 撤回
  - 聊天记录筛选
  - 附件管理

### 3. 输入区结构不是微信电脑端那套
- 当前桌面输入区是：
  - 左侧 `+` 菜单里再展开“发送图片 / 发送文件 / 发送收藏”
  - 独立麦克风按钮
  - 独立发送按钮
- 微信 `4.1.8.21` 已公开把桌面聊天能力收口成输入框底部的一排直接入口：
  - 表情
  - 发送收藏
  - 发送文件
  - 截图
  - 聊天记录
  - 语音输入文字
- 也就是说我们不是缺按钮数量，而是按钮摆放方式、默认路径、用户手部路径都不一样。

### 4. 缺截图工具，且没有微信式截后流程
- 现在桌面群聊可发图片文件，但没有聊天内截图入口。
- 微信 `4.0.2` 与 `4.1.0` 的公开信息都强调了截图增强：
  - 截图一键打码
  - 截图可置顶桌面
- Web 端不一定能 1:1 做“置顶桌面”，但聊天输入工具栏必须先补“截图并发送”的主路径，否则桌面味道不对。

### 5. 批量消息操作还没到微信 4.1.8 水平
- 现在桌面群聊多选后支持：
  - 收藏
  - 逐条转发
  - 删除
- 但仍缺：
  - 合并转发
  - 批量消息撤回
  - 批量图片 / 文件 / 未来视频消息的统一回收逻辑
- 微信 `4.1.8` 公开口径已经覆盖“撤回多选转发和批量发送的图片 / 视频 / 文件”。

### 6. 收藏与提醒仍然是本机 localStorage 心智，不像微信账号级能力
- 当前桌面收藏存在 `desktop-favorites-storage.ts`，直接写本地 `localStorage`。
- 当前消息提醒存在 `local-chat-message-actions.ts`，也是本地 `localStorage`。
- 微信里的“收藏”是账号级、多端同步能力；桌面群聊里“发送收藏”只是调起这个全局能力，不是当前浏览器私有状态。
- 这类能力如果继续停留在本地状态，用户会立刻觉得“像 UI，不像微信”。

### 7. 聊天记录检索 UI 像了，但底层仍不是全量搜索
- 当前右侧聊天记录面板虽然有：
  - 关键词
  - 类型
  - 日期
  - 群成员
- 但底层实现仍是：
  - 先拉最近 `80` 条
  - 继续增量拉取
  - 前端本地过滤
  - 最多只展示 `60` 条可见结果
- 微信的“按群成员 / 日期查聊天记录”心智是对整段历史做检索，不是“在最近一小段记录里筛”。

## P1：高优先级收口项

### 8. 群聊右上角和输入区的功能分层不对
- 现在顶部右上角是：
  - 聊天记录
  - 通话
  - 更多
- 当前线程头部下面又额外挂了一个带“群接龙”按钮的群公告条。
- 微信电脑端更像是：
  - 头部只放会话级动作
  - 输入区承载发消息相关动作
  - 群资料侧栏承载群协作设置
- 结论：
  - “群接龙”不该长期以大 CTA 挂在线程顶部
  - 更应回收到群资料面板与消息卡片流里

### 9. 还缺独立聊天窗口能力
- 当前只有图片查看器支持独立工作流。
- 微信 `4.1.0` 已公开支持拖拽拉出独立聊天窗口，`4.0.6` 时代也已存在独立聊天窗口长期心智。
- 对电脑端来说，这是非常核心的桌面多任务能力，不是锦上添花。

### 10. 群成员面板还不够像微信的“群管理工具”
- 当前右侧群成员区已经有成员宫格、更多成员、添加、移除。
- 但还缺更微信式的几个关键动作：
  - 成员搜索
  - 群主 / 管理员在浏览层直接可见
  - 更清楚的成员排序
  - 更像微信的完整成员浏览弹层
- 现在更像“数据维护器”，不像“微信桌面群资料页”。

### 11. 缺群聊备注
- 微信 Windows 在 2020-12-04 的 `3.1.0` 测试版就已支持给群聊设置备注。
- 当前项目只有：
  - 群聊名称
  - 我在本群的昵称
- 还缺“我对这个群的个人备注名”这一层。

## P2：细节层面的微信化

### 12. 语音输入只有功能，没有微信式快捷触发
- 当前桌面输入区有麦克风按钮，但没有微信 `4.1.8` 的：
  - Windows `Ctrl + Win`
  - macOS `fn`
- 对桌面用户来说，快捷键是微信习惯的一部分，不只是额外彩蛋。

### 13. 图片与多媒体工作流还不够桌面化
- 当前项目有图片查看和聊天图片独立窗口路由，但未对齐微信 `4.1.0` 的：
  - 同时打开多张图片
  - 更接近系统桌面的多窗口图片处理
- 如果后续补了视频消息，这块差异会更明显。

### 14. 视觉气质仍不像微信 PC
- 当前项目大量使用：
  - 大圆角
  - 明显卡片阴影
  - 偏暖色发送按钮
  - 更重的装饰性层次
- 微信 `4.1.8` 的公开表现更接近：
  - 白灰主色
  - 更弱投影
  - 更强系统感
  - 输入区和主面板一体化

## 该删掉或收口的“自创操作”

### 1. 不要把微信有的功能做成项目自定义入口
- “发送收藏”不该藏在 `+` 二级菜单里。
- “聊天记录”不该只放在头部历史按钮里，也应该纳入输入区桌面工具条心智。
- “群接龙”不该长期做成公告条右侧主 CTA。

### 2. 不要继续放大本地态假象
- 收藏、提醒、部分消息状态仍然放在本地存储，这会让用户以为桌面群聊是 demo 而不是正式聊天端。

## 直接照抄微信的实施顺序

## Phase 1：先改布局和操作顺序，不先堆功能
- 目标：先把用户手部路径改成微信电脑版。
- 要做：
  - 重做桌面输入区工具栏，改成底部显式入口
  - 把“发送收藏 / 发送文件 / 截图 / 聊天记录 / 语音输入”调整到微信式排布
  - 收掉线程顶部“群接龙”主按钮，只保留公告摘要
  - 统一头部右上角按钮职责，只保留会话级动作

## Phase 2：补齐桌面群聊最核心能力缺口
- 目标：补齐微信当前最显眼的代差。
- 要做：
  - 真正群语音 / 群视频能力
  - 群视频共享屏幕
  - 视频消息上传、发送、渲染、预览、倍速播放
  - 截图并发送链路
  - 多选消息合并转发 / 批量撤回

## Phase 3：把“看起来像”升级成“用起来像”
- 目标：把微信的检索、同步和桌面效率动作补完整。
- 要做：
  - 收藏中心服务端化，至少做到账号级持久化
  - 消息提醒去本地化，最少要做到同实例稳定持久
  - 聊天记录检索改为服务端全量搜索
  - 群成员完整浏览、搜索、角色标记
  - 群聊备注

## Phase 4：补桌面专属完成度
- 目标：做出微信 PC 的多窗口和工具感。
- 要做：
  - 独立聊天窗口
  - 图片多窗口工作流
  - 语音输入快捷键
  - 白灰桌面视觉 token 收口

## 今日结论
- 现在的桌面群聊已经有七成微信能力，但剩下三成正好都是最容易露馅的部分。
- 如果目标真是“完全抄微信电脑版群聊”，优先级不是再加零散设置，而是先改：
  - 输入区结构
  - 真群通话
  - 视频消息
  - 截图
  - 批量操作
  - 服务端检索与收藏同步
