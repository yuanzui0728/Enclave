# Web 端电脑版聊天页对标微信差异与微信化规划

## 任务目标
- 盘清 `apps/app` 当前 Web 端电脑版聊天页，和微信电脑端聊天页在功能、操作、信息架构上的差异。
- 给出一条“尽量照着微信做”的落地路线，优先复制用户最有肌肉记忆的部分。
- 本轮只做规划，不改业务代码。

## 参考基线

### 本地代码现状
- 会话工作区：`apps/app/src/features/desktop/chat/desktop-chat-workspace.tsx`
- 单聊线程：`apps/app/src/features/chat/conversation-thread-panel.tsx`
- 群聊线程：`apps/app/src/features/chat/group-chat-thread-panel.tsx`
- 消息列表：`apps/app/src/components/chat-message-list.tsx`
- 输入框：`apps/app/src/components/chat-composer.tsx`
- 右侧聊天信息面板：`apps/app/src/features/desktop/chat/desktop-chat-details-panel.tsx`
- 右侧聊天记录面板：`apps/app/src/features/desktop/chat/desktop-chat-history-panel.tsx`

### 微信公开口径
- 2026-03-14，新浪转载 IT 之家：微信 `4.1.8 for Mac / Windows` 已支持“语音输入文字”。
- 2026-03-19，搜狐转引“微信派”：微信 `4.1.8` 电脑端支持 `Ctrl + Win` 语音输入，并支持撤回多选转发、批量发送的图片 / 视频 / 文件。
- 2025-11-01，中新网总结鸿蒙电脑版微信新能力，并明确说明近期 MacOS / Windows 端新增功能已同步发布，包括：
  - 聊天记录备份到电脑
  - 右键聊天内容设置提醒
  - 文件 / 视频未下载也能直接转发、多选、引用
  - 收藏内容可直接发送
  - 群聊视频通话
  - 群聊对文字、图片、视频、表情的精准引用
- 长期稳定桌面交互基线，可参考 2017 年爱范儿对 Windows 微信功能盘点：
  - 拖拽文件到聊天框发送
  - 聊天消息右键的引用 / 多选 / 转发 / 收藏
  - 聊天记录备份与恢复
  - 独立聊天窗口 / 置顶窗口

### 规划原则
- 信息架构、入口顺序、右键菜单、批处理流、侧栏结构，优先按微信电脑端去做。
- 不复制微信品牌资产、Logo、版权图片；只复制交互和产品结构。
- 如果某个微信能力当前后端没链路，首版至少要把入口形态和工作流做得一致，不能再停留在“占位提示”。

## 当前现状

### 已经有的能力
- 左侧会话列表、会话搜索、快捷加号菜单已经成型。
- 单聊支持文字、表情、图片、文件发送，输入框支持粘贴文件，支持语音转写入口。
- 群聊支持 `@` 候选、回复引用、图片 / 文件 / 名片 / 位置卡片发送。
- 右侧已经有两个面板雏形：
  - `聊天记录`
  - `聊天信息`
- 也已经有独立页：
  - `桌面聊天文件`：`/desktop/chat-files`
  - `桌面聊天记录管理`：`/desktop/chat-history`

### 当前明显不是微信的地方
- 聊天页头部和面板整体仍然是项目自定义风格，微信感不够。
- 核心操作大量停留在“半成品”状态，尤其是右键、批处理、真实同步设置。

## 差异清单

### P0：必须先补，用户一上手就会觉得“不像微信”的差异

#### 1. 消息右键菜单差得很大
- 当前桌面端右键菜单只在群聊消息里出现，并且只有：
  - `回复消息`
  - `复制消息`
  - `复制发送者`
- 代码位置：
  - `apps/app/src/components/chat-message-list.tsx:51`
  - `apps/app/src/features/chat/group-message-context-menu.tsx:57`
- 直接差异：
  - 单聊消息没有右键菜单
  - 群聊右键没有多选、转发、收藏、撤回、删除、提醒、翻译、另存、打开文件位置等微信高频动作
  - 图片、文件、表情、引用消息没有按类型给出对应菜单

#### 2. 通话入口还是占位，不是微信可用能力
- 顶部通话按钮点下去只是 toast：
  - `视频通话入口已保留，真实通话能力后续补齐`
  - `语音通话入口已保留，真实通话能力后续补齐`
- 代码位置：
  - `apps/app/src/features/desktop/chat/desktop-chat-workspace.tsx:240`
- 这和微信电脑端“能发起语音 / 视频，群里还能视频通话”的感知差很大。

#### 3. 单聊没有“回复引用”闭环
- 群聊已有回复草稿和消息跳转。
- 单聊线程 `ConversationThreadPanel` 仍然没有把回复引用接进去。
- 结果就是单聊无法复制微信现在最常用的“回复某条消息”操作习惯。

#### 4. 免打扰 / 显示群成员昵称还是本地假状态
- 单聊聊天信息里的 `消息免打扰` 只是写 `localStorage`，没有调用现成后端接口。
- 群聊里的 `消息免打扰`、`显示群成员昵称` 也是本地偏好，不是群设置真实同步。
- 代码位置：
  - `apps/app/src/features/chat-details/chat-detail-preferences.ts`
  - `apps/app/src/features/desktop/chat/desktop-chat-details-panel.tsx:310`
  - `apps/app/src/features/desktop/chat/desktop-chat-details-panel.tsx:659`
- 但仓库里其实已经有接口：
  - `setConversationMuted`
  - `updateGroupPreferences`
  - `hideConversation`
  - `hideGroup`
- 这说明不是后端没能力，而是桌面端没接。

#### 5. 会话列表缺少微信式右键操作
- 当前左侧会话列表只能点进会话，没有右键菜单。
- 微信电脑端用户最常用的会话级动作基本都应该在这里完成：
  - 置顶 / 取消置顶
  - 免打扰 / 取消免打扰
  - 标为未读 / 标为已读
  - 隐藏聊天
  - 删除聊天
  - 清空聊天记录
- 代码位置：
  - `apps/app/src/features/desktop/chat/desktop-chat-workspace.tsx:324`

#### 6. 群聊信息面板还有明显“占位实现”
- 移除成员只是提示“入口已保留”。
- 群名、群公告、我在本群昵称还是 `window.prompt`。
- 代码位置：
  - `apps/app/src/features/desktop/chat/desktop-chat-details-panel.tsx:551`
  - `apps/app/src/features/desktop/chat/desktop-chat-details-panel.tsx:603`
  - `apps/app/src/features/desktop/chat/desktop-chat-details-panel.tsx:673`
- 这和微信电脑端成熟的面板交互差距很大。

#### 7. 桌面端没有拖拽发送文件
- 当前输入框支持粘贴文件，但没有拖拽文件到聊天区直接发送。
- 微信桌面端这是最典型的办公场景操作。
- 当前代码只看到粘贴处理：
  - `apps/app/src/components/chat-composer.tsx:801`

### P1：已经影响完成度，但可排在主链路之后

#### 8. 聊天记录侧栏能力太浅
- 当前右侧聊天记录面板是“拉全量消息到前端，本地搜索，只显示最多 60 条”。
- 代码位置：
  - `apps/app/src/features/desktop/chat/desktop-chat-history-panel.tsx:28`
  - `apps/app/src/features/desktop/chat/desktop-chat-history-panel.tsx:64`
- 和微信相比还缺：
  - 按日期 / 发送者 / 类型筛选
  - 图片 / 文件 / 链接 / 小程序等分类检索
  - 更稳定的“在原会话中定位”
  - 更早记录分页

#### 9. 聊天文件入口还是“独立管理页”，不是微信式就地工作流
- 现在的 `桌面聊天文件` 页是另一套全页工作区。
- 还能看出备注“后续再补跨会话筛选与搜索 / 批量操作和回跳原消息”。
- 代码位置：
  - `apps/app/src/routes/desktop-chat-files-page.tsx:117`
  - `apps/app/src/routes/desktop-chat-files-page.tsx:202`
- 微信更像是在聊天上下文里直接处理文件，而不是跳出当前心流。

#### 10. 引用能力不完整
- 群聊已有回复引用，但单聊没有。
- 群聊右键也没有针对图片 / 视频 / 表情的完整引用菜单。
- 微信里这些已经是统一能力，不该只在群聊半接通。

#### 11. 收藏链路没接到聊天页
- 微信桌面端能把聊天内容收藏，也能把收藏内容直接发回聊天。
- 当前项目有 `收藏` 一级工作区，但聊天页没有消息级收藏，也没有“从收藏发回聊天”的工作流。

#### 12. 没有聊天内容提醒
- 微信公开口径里已经有“右键聊天内容设置提醒”。
- 当前聊天页完全没有这个能力。

### P2：更偏“像不像微信”的细节差异

#### 13. 表面视觉还是项目自定义，不是微信桌面端
- 现在大量使用米色渐变、暖色阴影、卡片圆角。
- 微信桌面端更克制、更平、更白灰。
- 这会导致即使功能补齐，第一眼仍然不像。

#### 14. 缺少独立聊天窗口 / 窗口置顶能力
- 这是微信桌面端非常典型的效率能力。
- 当前没有把会话拆成子窗口，也没有聊天窗口置顶。

#### 15. 缺少“聊天记录备份到电脑”一类桌面专属能力
- 微信把电脑端当作资料管理端。
- 当前聊天页和桌面壳之间还没有这类强桌面能力的串联。

## 建议的“完全按微信抄”实施顺序

### Phase 1：先把会话列表和头部操作抄到位
目标：用户一进聊天页，第一眼和第一手操作就接近微信。

要做的事：
- 给左侧会话卡片补右键菜单。
- 补齐会话级动作：
  - 置顶
  - 免打扰
  - 标记已读 / 未读
  - 隐藏聊天
  - 删除 / 清空
- 把顶部三按钮继续保留为：
  - `聊天记录`
  - `通话`
  - `更多`
- 但通话不能再只是提示，至少要做完整弹层和状态流。

优先文件：
- `apps/app/src/features/desktop/chat/desktop-chat-workspace.tsx`
- 新增 `apps/app/src/features/desktop/chat/desktop-conversation-context-menu.tsx`
- `packages/contracts/src/client.ts` 只补当前桌面端缺的调用封装时再动

### Phase 2：把消息区操作抄成微信的主战场
目标：右键、引用、转发、多选这些高频动作一次补齐。

要做的事：
- 把当前群聊专用右键菜单升级成“桌面消息通用右键菜单”。
- 单聊、群聊统一支持：
  - 回复
  - 复制
  - 多选
  - 转发
  - 收藏
  - 删除
  - 撤回
  - 设置提醒
- 附件类型差异化菜单：
  - 图片：另存、打开原图、转发、收藏、引用
  - 文件：打开、另存、打开所在位置、转发、收藏、引用
  - 表情：添加到表情、转发、引用
- 输入区补拖拽发送。
- 单聊接入回复引用，和群聊统一。

优先文件：
- `apps/app/src/components/chat-message-list.tsx`
- `apps/app/src/features/chat/group-message-context-menu.tsx`
- 新增 `apps/app/src/features/desktop/chat/desktop-message-context-menu.tsx`
- 新增 `apps/app/src/features/desktop/chat/desktop-message-selection-bar.tsx`
- `apps/app/src/features/chat/conversation-thread-panel.tsx`
- `apps/app/src/features/chat/group-chat-thread-panel.tsx`
- `apps/app/src/components/chat-composer.tsx`

### Phase 3：把右侧聊天信息面板从“能看”做成“能管”
目标：复制微信电脑端“三个点”后的主信息架构。

要做的事：
- 单聊面板接通真实后端：
  - `setConversationMuted`
  - `setConversationPinned`
  - `hideConversation`
- 群聊面板接通真实后端：
  - `updateGroupPreferences`
  - `setGroupPinned`
  - `hideGroup`
  - `removeGroupMember`
- 去掉 `window.prompt`，改成微信式行内弹层 / 小弹窗。
- 群成员区域改成更接近微信的头像宫格与 `+ / -` 入口。

优先文件：
- `apps/app/src/features/desktop/chat/desktop-chat-details-panel.tsx`
- `apps/app/src/features/desktop/chat/desktop-group-member-picker.tsx`
- 可参考移动端已接通逻辑：
  - `apps/app/src/routes/group-chat-details-page.tsx`
  - `apps/app/src/routes/contacts-page.tsx`

### Phase 4：把聊天记录 / 聊天文件做成微信式上下文能力
目标：不再像两个独立业务页，而是聊天工作区的一部分。

要做的事：
- 右侧聊天记录面板增强为：
  - 分类检索
  - 时间筛选
  - 发送者筛选
  - 更老消息分页
- 聊天文件支持：
  - 当前会话上下文过滤
  - 多选
  - 转发
  - 收藏
  - 回到原消息
  - 打开 / 下载 / 另存
- 如果要更贴近微信，可把“聊天文件”先收进聊天信息面板二级入口，再决定是否保留独立页。

优先文件：
- `apps/app/src/features/desktop/chat/desktop-chat-history-panel.tsx`
- `apps/app/src/routes/desktop-chat-files-page.tsx`
- `apps/app/src/routes/desktop-chat-history-page.tsx`

### Phase 5：最后补桌面端独有高级能力
目标：从“像微信”提升到“真的像电脑版微信”。

要做的事：
- 真实语音 / 视频通话
- 群视频通话
- 独立聊天窗口
- 聊天窗口置顶
- 聊天记录备份到电脑
- 收藏内容直接回发聊天
- 截图时隐藏聊天窗口
- 文件未下载也能转发 / 引用

这阶段会牵涉：
- `apps/desktop`
- 桌面壳能力
- 可能新增 RTC / 本地文件管理 / 备份介质链路

## 我建议的实际开工顺序
- 第一轮只做 `Phase 1 + Phase 2`。
- 第二轮做 `Phase 3`，把现在桌面端“假设置”全部换成真实同步。
- 第三轮做 `Phase 4`，把聊天记录和文件能力补深。
- 第四轮再碰通话、独立窗口、备份。

原因很直接：
- 用户最强烈感知的是右键菜单、会话菜单、引用、多选、拖拽发送。
- 这些补完，聊天页就已经会非常像微信。
- 通话和备份很重，应该后置，不该阻塞主聊天心流的微信化。

## 验收口径
- 左侧会话列表支持微信式右键操作。
- 单聊、群聊消息都支持统一右键菜单和多选模式。
- 单聊、群聊都支持回复引用。
- 拖文件到聊天区可直接发送。
- 三个点后的聊天信息面板不再依赖本地假状态，改为真实接口同步。
- 群聊成员移除、群昵称、群公告不再用 `window.prompt` 和占位提示。
- 右侧聊天记录至少支持更完整搜索和稳定回跳原消息。
- 页面视觉收平，整体第一眼像微信电脑端，而不是当前项目自定义暖色卡片风。

## 结论
- 现在的 Web 端电脑版聊天页，已经有“能聊天”的骨架，但离微信电脑端还差一整层“操作系统级肌肉记忆”。
- 真正该优先抄的，不是继续加零散功能，而是先把：
  - 会话右键
  - 消息右键
  - 回复 / 多选 / 转发 / 收藏
  - 拖拽发送
  - 真实同步设置
  这五件事补齐。
- 只要这五件事到位，聊天页主观感受会立刻从“像一个自定义 IM”变成“就是微信那套桌面工作流”。
