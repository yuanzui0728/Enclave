# Web 端手机版聊天页对标微信差异规划

## 任务目标
- 对标 `apps/app` 当前 Web 手机版单聊页，梳理它和微信手机端聊天页在功能、操作和表层视觉上的差异。
- 目标不是“做一个能聊天的页面”，而是尽量把聊天页做成用户一眼看上去、上手操作起来都像微信手机端。
- 本次规划优先覆盖单聊主链路：
  - `/chat/$conversationId`
  - `/chat/$conversationId/details`
  - `/chat/$conversationId/search`
- 群聊只在“可复用实现”层面作为参考，因为当前群聊里已经有一部分回复能力可以拿来复用。

## 参考基线

### 公开信息确认到的部分
- 微信 iPhone App Store 页面当前仍明确强调这些聊天能力：
  - 支持发送 `文本 / 图片 / 视频 / 语音消息`
  - 支持 `语音和视频聊天`
  - 支持 `实时位置共享`
  - 支持 `聊天室墙纸自定义`
  - 支持 `消息提醒自定义`
- 2025-04-03 的公开报道转述了微信派发布的更新，能确认这些近年的聊天能力：
  - `引用回复` 已支持继续回 `语音 / 表情包`
  - `聊天记录支持分类搜索`
  - `长按消息支持收藏`
  - `消息撤回后支持重新编辑和发送`

### 需要明确标注的推断
- 微信手机端聊天页的一些细节交互，例如：
  - 输入栏左侧 `语音 / 键盘` 切换
  - `按住说话`
  - 消息长按弹出操作菜单
  - 时间分隔条不是每条消息都显示
- 这些没有在公开文案里逐项列清，但属于微信手机端长期稳定、且用户感知极强的基础交互，本规划按“当前主流微信手机端习惯”处理。

## 当前现状

### 1. 聊天页表层样式还不像微信手机端
- `apps/app/src/features/chat/conversation-thread-panel.tsx`
  - 移动端单聊主背景仍是米白渐变，不是微信手机端更接近 `#ededed` 的平面聊天底色。
  - 单聊头部默认副标题会显示 `连接顺畅`，这不是微信聊天页的常态信息层。
- `apps/app/src/features/chat/mobile-chat-thread-header.tsx`
  - 顶部栏是项目自定义的“圆角按钮 + 玻璃感”风格，不是微信手机端更扁平、更紧的导航栏。
- `apps/app/src/components/chat-composer.tsx`
  - 输入区外层使用了圆角胶囊、投影、暖色按钮和卡片式面板，整体更像项目 UI，不像微信底部工具栏。

### 2. 消息流展示还停留在“能看”，没到微信式聊天页
- `apps/app/src/components/chat-message-list.tsx`
  - 现在每条消息上方都会显示时间。
  - 微信手机端通常是“按时间段插入时间分隔条”，不是每条都带时间。
- 当前图片消息只是内联 `<img>`，没有微信式点开全屏预览、左右切换、保存、转发入口。
- 当前文件、位置、名片卡片是可发送的，但视觉和操作仍偏“组件卡片”，不是微信消息卡片节奏。

### 3. 输入栏差距很大，这是最明显的不像微信的地方
- `apps/app/src/components/chat-composer.tsx`
  - 移动端左侧是一个独立 `Mic` 按钮，点击后打开“语音转文字”弹层。
  - 这不是微信的 `语音 / 键盘` 切换 + `按住说话` 语音消息模式。
  - 当前输入框是单行 `<input>`，不是微信那种可增长的多行输入框。
  - 右侧只有 `表情` 和 `+`，但 `+` 面板的布局、图标风格、翻页方式和项目文案都还不是微信。
  - 文本非空时显示橙色“发送”按钮，视觉也不是微信的绿色发送按钮体系。
- 现有语音能力本质是“浏览器录音 -> AI 转写 -> 插入文本”，不是微信原生的语音消息发送链路。

### 4. 消息操作几乎没有，微信高频操作缺口最大
- `apps/app/src/components/chat-message-list.tsx`
  - 右键/上下文菜单只在 `桌面端 + 群聊` 下启用。
  - 移动端没有长按消息菜单。
  - 单聊没有接入 `回复消息`。
- 也就是说当前移动端单聊缺这些微信高频动作：
  - `复制`
  - `回复`
  - `转发`
  - `收藏`
  - `多选`
  - `撤回`
  - `删除`
  - `提醒`
- 其中一部分 UI 可以先做，另一部分需要补消息模型和接口。

### 5. 单聊其实已经有“引用回复显示能力”，但没接完整
- `apps/app/src/components/chat-message-list.tsx`
  - 已经能渲染回复引用块。
- `apps/app/src/features/chat/group-chat-thread-panel.tsx`
  - 群聊已经有 `replyDraft`、`encodeChatReplyText`、`ReplyPreviewBar` 这整套回复草稿能力。
- `apps/app/src/features/chat/conversation-thread-panel.tsx`
  - 单聊没有把这套逻辑接进来。
- 这说明单聊“回复消息”不是从零做，主要是移动端消息长按入口没做、单聊线程状态没补。

### 6. 查找聊天记录只是最基础文本搜索，离微信分类搜索还远
- `apps/app/src/routes/chat-message-search-page.tsx`
  - 目前只支持按 `message.text` 做本地关键字过滤。
  - 没有微信式分类入口，例如：
    - 日期
    - 图片与视频
    - 文件
    - 链接
    - 语音
    - 交易/提醒类消息

### 7. 聊天信息页已有一部分设置，但还不是完整微信口径
- `apps/app/src/routes/chat-details-page.tsx`
  - 目前已有：
    - `查找聊天记录`
    - `消息免打扰`
    - `置顶聊天`
    - `强提醒`
    - `保存到通讯录`
    - `设置当前聊天背景`
    - `清空聊天记录`
    - `投诉`
    - `加入黑名单`
- 这些能力能支撑一部分“微信式聊天信息页”，但和聊天主页面直接相关的操作链路还没接起来：
  - 从消息长按进入回复/收藏/转发
  - 从图片进入查看器
  - 从输入栏进入语音消息、更多面板

### 8. 当前消息契约不支持很多微信手机端聊天能力
- `packages/contracts/src/chat.ts`
  - 当前消息类型只有：
    - `text`
    - `system`
    - `proactive`
    - `sticker`
    - `image`
    - `file`
    - `contact_card`
    - `location_card`
- `packages/contracts/src/ws.ts`
  - 当前单聊发送协议也只覆盖上述类型。
- 也就是这些能力现在没有契约基础：
  - `voice`
  - `video`
  - `forwarded bundle`
  - `favorite state`
  - `recall state`
  - `message reminder`
  - `live location`

## 差异清单

### P0：不改就绝对不像微信
- 单聊页面底色、头部、底部输入栏全部需要微信化收平。
- 去掉默认副标题 `连接顺畅`，只在必要时临时展示状态信息。
- 时间显示改成“时间分隔条”，不再每条消息都显示时间。
- 输入框改成微信式底栏结构：
  - 左侧 `语音 / 键盘` 切换
  - 中间多行输入框
  - 右侧 `表情`
  - 文本为空时显示 `+`
  - 文本非空时显示绿色 `发送`
- `+` 面板改成微信式宫格，不再是当前项目风格的渐变卡片。

### P1：微信高频操作，用户马上会点
- 移动端长按消息菜单。
- 单聊 `回复消息`。
- 图片点开预览。
- 表情面板和加号面板的开合节奏、遮罩、键盘切换行为对齐微信。
- 输入框自动增长、高度封顶、超出后内部滚动。

### P2：微信聊天页的核心效率能力
- `转发`
- `收藏`
- `多选`
- `删除`
- `撤回`
- `撤回后重新编辑`
- `消息设提醒`
- `分类搜索`

### P3：业务能力型功能
- `语音消息`
- `视频消息`
- `实时位置共享`
- `语音通话 / 视频通话`
- `红包 / 转账`
- 这一层已经不只是“抄页面”，而是要补消息协议、上传链路、播放链路、甚至后端状态机。

## 目标口径

### 1. 先抄“表面和操作节奏”，不是先补全所有后端业务
- 用户说的“完全抄袭”，在工程上应该拆成两层：
  - 第一层：视觉、结构、手势、操作路径 1:1 像微信
  - 第二层：业务型消息类型也真的能用
- 第一层应该先做完，否则页面再多功能也不会像微信。

### 2. 单聊页首版目标
- 打开单聊后，用户第一眼就应该觉得像微信手机端：
  - 顶部导航栏像
  - 消息区像
  - 输入栏像
  - 长按消息的反馈像
  - 表情和加号面板像
- 首版即便还没补 `红包 / 转账 / 实时位置共享`，也要先把聊天主链路做成微信味道。

### 3. 单聊和群聊共享的基础件要统一
- 头部
- 时间分隔条
- 消息长按菜单
- 图片查看器
- 输入栏框架
- 表情面板
- `+` 面板
- 这样后续群聊微信化时不会重复拆。

## 推荐实现方案

### Phase 1：先把聊天页表皮改成微信
目标：解决“一眼不像微信”的问题。

改造点：
- `apps/app/src/features/chat/conversation-thread-panel.tsx`
  - 单聊移动端背景改成微信手机端灰底。
  - 去掉默认 `连接顺畅` 副标题。
  - 状态提示改成更微信式的临时态展示。
- `apps/app/src/features/chat/mobile-chat-thread-header.tsx`
  - 改成更接近微信手机端的导航栏：
    - 左侧返回
    - 中间标题
    - 右侧三个点
  - 不保留当前圆角悬浮按钮风格。
- `apps/app/src/components/chat-message-list.tsx`
  - 把“每条消息时间”改成“按时间切片插入分隔条”。
  - 调整消息区留白、头像间距、气泡圆角、系统消息样式。
- `apps/app/src/components/chat-composer.tsx`
  - 重做移动端底部结构，变成微信式工具栏。

验收口径：
- 页面上不再出现现在这套暖色渐变、玻璃按钮、浮卡感。
- 单聊页和群聊页的移动端聊天背景保持同一套微信基底。

### Phase 2：补齐输入栏的微信式核心操作
目标：让输入行为像微信，不再像“AI 转写输入框”。

改造点：
- `apps/app/src/components/chat-composer.tsx`
  - 左侧改成 `语音 / 键盘` 切换按钮，而不是直接弹语音转写面板。
  - 文本态使用多行 `textarea`，支持自动增高。
  - 空文本时显示 `+`，非空时显示绿色发送。
  - 表情入口与加号入口切换时，键盘、表情面板、加号面板互斥。
- `apps/app/src/components/mobile-chat-plus-panel.tsx`
  - 面板视觉改成微信式浅灰底宫格。
  - 首屏入口建议调整为：
    - `照片`
    - `拍摄`
    - `位置`
    - `名片`
    - `文件`
  - 先不放当前做不了的红包/转账，但保留后续扩展位。
- `apps/app/src/components/mobile-speech-input-sheet.tsx`
  - 不再作为主语音入口。
  - 首版可降级为 `语音转文字` 次级入口，收进 `+` 面板或输入增强入口。

验收口径：
- 用户可以用微信式底栏节奏完成输入、开表情、开加号、发送文本。
- 当前“麦克风录音转写”的存在感显著下降，不再主导聊天输入体验。

### Phase 3：补齐单聊回复链路
目标：把单聊的“长按回复”做成微信高频操作。

核心复用：
- 直接复用群聊已存在的回复数据结构和展示方式：
  - `apps/app/src/features/chat/group-chat-thread-panel.tsx`
  - `apps/app/src/lib/chat-text.ts`
  - `apps/app/src/components/chat-message-list.tsx`
  - `apps/app/src/components/chat-composer.tsx`

改造点：
- `apps/app/src/features/chat/conversation-thread-panel.tsx`
  - 增加 `replyDraft` 状态。
  - 接入 `replyPreview` 和 `onCancelReply`。
  - 发送文本、图片、文件、名片、位置时都支持带引用上下文。
- `apps/app/src/components/chat-message-list.tsx`
  - 移动端支持长按消息。
  - 长按菜单首批加入：
    - `回复`
    - `复制`
    - `删除`
    - `更多`

验收口径：
- 单聊长按一条消息后，可以进入回复态。
- 回复态在输入区上方显示被回复内容，发送后消息区能显示引用块。

### Phase 4：补消息长按菜单和多步操作
目标：把微信用户最常点的消息菜单接出来。

建议首批菜单：
- `回复`
- `复制`
- `转发`
- `收藏`
- `删除`
- `多选`

二批菜单：
- `撤回`
- `提醒`
- `翻译`
- `提取文字`

实现建议：
- 新增移动端 Action Sheet，而不是复用桌面端右键菜单。
- 图片、文件、位置、名片要有类型化菜单，不是所有消息都共用同一份按钮。

### Phase 5：做图片查看器和附件消息闭环
目标：把“图片消息”从可显示升级成可操作。

改造点：
- `apps/app/src/components/chat-message-list.tsx`
  - 图片消息可点开。
- 建议新增：
  - `apps/app/src/features/chat/mobile-image-viewer.tsx`
- 能力首批包括：
  - 单图全屏预览
  - 左右滑动切换
  - 保存
  - 转发入口占位

### Phase 6：补聊天记录分类搜索
目标：把 `查找聊天记录` 从文本搜索升级成微信式分类搜索入口。

改造点：
- `apps/app/src/routes/chat-message-search-page.tsx`
  - 先改成微信式搜索入口页。
  - 顶部搜索框下加入分类入口。
- 首批分类建议：
  - `日期`
  - `图片与视频`
  - `文件`
  - `链接`
- 当前没有 `语音消息` 契约，语音分类放到后续阶段。

## 数据与接口补充规划

### 现有模型足够支撑的能力
- 页面微信化表皮
- 长按 `复制`
- 单聊 `回复`
- 图片查看器前端壳
- 输入栏重做
- 加号面板重做
- 文本搜索升级为分类入口壳

### 需要补消息模型或接口的能力
- `语音消息`
  - 新消息类型 `voice`
  - 音频附件元数据
  - 录音上传和播放链路
- `视频消息`
  - 新消息类型 `video`
- `收藏`
  - 需要消息级收藏记录
- `转发`
  - 需要消息复制/合并转发接口
- `撤回`
  - 需要消息状态和撤回权限
- `撤回后重新编辑`
  - 需要保留最近一次撤回文本草稿
- `提醒`
  - 需要消息提醒记录或本地提醒模型
- `实时位置共享`
  - 当前 `location_card` 只是静态位置卡片，不是实时共享

### 建议的数据改造顺序
1. 不动契约，先完成 Phase 1-3。
2. 再补 `收藏 / 转发 / 撤回` 这类消息操作接口。
3. 最后补 `voice / video / live location` 这类附件型消息。

## 明确不建议现在就混在一起做的项
- `红包 / 转账`
- `真实语音通话 / 视频通话`
- `实时位置共享`
- 这些一旦做，不是“抄聊天页”，而是额外一条完整业务线。
- 如果当前目标只是“让 Web 手机版聊天页像微信”，应该排在后面。

## 推荐落地顺序
1. 先改移动端聊天表皮和输入栏框架。
2. 接单聊回复和移动端长按菜单。
3. 补图片查看器。
4. 升级加号面板和表情面板节奏。
5. 重做聊天记录搜索入口。
6. 再决定要不要继续做 `收藏 / 转发 / 撤回 / 语音消息`。

## 最终验收口径
- 打开 `/chat/$conversationId` 后，整体观感应立即接近微信手机端，而不是现在的项目自定义聊天页。
- 用户能用微信式底栏节奏完成：
  - 输入文字
  - 打开表情
  - 打开加号面板
  - 长按消息
  - 回复消息
  - 查看图片
- 消息列表不再逐条显示时间，而是按时间段插入分隔条。
- 聊天详情页和查找聊天记录页继续保留现有路由，但入口与视觉语言向微信手机端收拢。
