# Web 端电脑版通讯录页与聊天页对标微信电脑端差异规划

## 任务目标
- 盘清当前 Web 端电脑版 `通讯录` 和 `聊天页`，与微信电脑端在功能、操作、信息架构上的真实差异。
- 明确哪些能力已经有了，哪些地方还需要继续“照着微信抄”。
- 输出一条可直接开工的收口顺序，优先补用户最能感知到的差异。

## 微信电脑端参考基线

### 公开资料
- `2025-03-31`，IT 之家：微信 PC `4.0.3` 已支持 `发表朋友圈 / 查看朋友圈相册 / 深浅色模式`，说明电脑端已经不只是“消息收发器”。
- `2025-04-13`，新浪转引 IT 之家：微信 PC `4.0.3.36` 已支持 `PC 端收款` 与 `历史版本冗余数据清理`。
- `2025-07-16`，WeChat Help Center：搜索聊天记录已支持按 `日期 / 群成员 / 文件 / 图片与视频 / 链接 / 小程序 / 音乐` 等条件检索。
- `2025-11-24`，WeChat Help Center：群聊支持 `群二维码` 作为邀请入口。
- `2026-01-23`，IT 之家：微信 Mac `4.1.7` 已支持 `群接龙 / 群管理 / 截图一键打码 / 转账收款 / 语音转文字`。

### 结论
- 微信电脑端现在的基准不是“有聊天列表就算像”。
- 真正要抄的是三件事：
  - 联系人资料页和聊天信息侧栏高度统一
  - 消息处理与聊天检索足够桌面化
  - 电脑端有明确的办公主端气质

## 当前已经有的，不要误判成缺失

  - 隐藏聊天
  - 清空聊天记录

  - 群聊

## 真实差异清单

### P0：必须先抄齐的差异

#### 1. 通讯录右侧详情仍然不像微信联系人资料页
- 当前 `contact-detail-pane` 还是“大头像 + 大标题 + 长描述 + 多段卡片”。
- 微信电脑端联系人详情更像紧凑资料页：
  - 顶部身份信息
  - 中部字段行
  - 内容入口行
  - 底部管理与危险操作
- 当前最大问题不是字段少，而是页面重心不对：
  - `bio`
  - 长说明文字
  - 世界角色标签
  - 这些都把页面拉回了“角色资料卡”。

#### 2. 通讯录还没有真正抄到微信的内容入口层
- 当前右侧只有：
  - `更多资料`
  - `朋友圈` 占位
  - `共同群聊` 文本
- 微信电脑端联系人详情更强调可点击入口：
  - 朋友圈 / 朋友相册
  - 共同群聊
  - 更多信息
  - 投诉 / 拉黑 / 删除联系人等管理项
- 这里最明显的缺口就是：
  - `朋友圈` 还没接
  - `共同群聊` 还不是明确入口流

#### 3. 通讯录左侧仍然混入了非微信心智
- 当前有 `世界角色` 入口和结果区，这属于隐界特有概念，不是微信通讯录语义。
- 如果目标是“完全照微信电脑端抄”，那它不能继续混在微信标准入口组中抢主心智。
- 建议处理：
  - 微信标准入口放前面
  - `世界角色` 下沉到单独分区或独立工作区

#### 4. 标签能力还停在占位
- 当前 `标签` 入口明确写着“待接入真实标签能力”。
- 微信电脑端通讯录里，标签不是装饰入口，而是联系人管理的高频路径。
- 这会导致用户一进通讯录就立刻感觉“像微信，但是假微信”。

#### 5. 联系人编辑方式不够微信
- 当前是右侧整块直接展开表单编辑。
- 微信更像逐项编辑或轻弹层编辑：
  - 改备注
  - 改标签
  - 改来源
- 当前交互更像后台表单，不像桌面 IM。

#### 6. 聊天页右上角“通话”还不是真桌面通话
- 当前点击语音 / 视频，本质还是跳 `/desktop/mobile` 做手机接力。
- 微信电脑端这里的用户预期是：
  - 真桌面语音
  - 真桌面视频
  - 群场景还能继续扩成桌面协作能力
- 这是最强感知差异之一。

#### 7. 聊天信息侧栏和通讯录详情仍然是两套人设
- 通讯录页已经有：
  - 备注
  - 地区
  - 来源
  - 标签
  - 共同群聊
- 聊天信息侧栏更偏聊天设置和管理。
- 微信电脑端不会让“通讯录里看到的人”和“聊天里看到的人”像两个模块拼起来的。
- 必须收口成：
  - 同一套顶部身份模块
  - 同一套字段顺序
  - 同一套危险操作顺序

#### 8. 聊天记录侧栏还只是“当前已加载消息里的本地搜索”
- 当前右侧聊天记录侧栏虽然有：
  - 关键词
  - 类型筛选
  - 日期筛选
  - 群成员筛选
- 但它只覆盖最近加载的一段消息，默认 80 条，展示最多 60 条。
- 和微信电脑端相比，差的不是按钮数，而是检索深度：
  - 不是全量历史
  - 不是服务端搜索
  - 不能稳定覆盖更早记录

#### 9. 聊天资料聚合还不够微信
- 微信电脑端现在对聊天资料的期待是：
  - 查找聊天记录
  - 聊天文件
  - 图片 / 视频 / 链接 / 小程序等分类入口
- 当前项目虽然有 `聊天记录侧栏`、`聊天文件页`，但还偏两套分散工作区。
- 还没形成微信那种“我在当前聊天上下文里继续查资料”的感觉。

### P1：已经接近，但还需要修到微信味更重

#### 10. 会话列表和主消息区视觉还是项目风格，不是微信风格
- 目前桌面聊天区、通讯录区都还用了暖色渐变、较重阴影、大圆角卡片。
- 微信电脑端更平、更冷、更克制。
- 这会让用户即使操作通了，第一眼也不会觉得像微信。

#### 11. 菜单文案和顺序还不够“微信肌肉记忆”
- 当前消息右键和会话右键已经很全。
- 现在的问题变成：
  - 文案顺序未必是微信习惯
  - 各消息类型的差异化动作还不够收敛
  - 某些动作的位置更像项目实现顺序，不像微信高频顺序

#### 12. 聊天头部按钮骨架是对的，但细节还不够桌面原生
- 现在是 `聊天记录 / 通话 / 更多`。
- 方向对，但还差：
  - 更微信式的按钮尺寸和留白
  - 更稳定的 hover / active 反馈
  - 头部与右侧面板联动节奏

### P2：微信电脑端的加分项，主链路稳定后再补

#### 13. 桌面专属效率能力还缺
- 独立聊天窗口
- 窗口置顶
- 截图置顶桌面
- 聊天记录备份 / 恢复 / 存储管理
- 更强的文件中转能力

#### 14. 通讯录电脑端的扩展分层还不够
- 微信电脑端除了联系人，还有更成熟的：
  - 公众号
  - 群聊
  - 标签
  - 企业微信等联系人类型分层
- 当前我们虽然有 `公众号` 和 `群聊`，但整体还是“功能入口列表 + 朋友详情”。

## 完全照微信抄的实施顺序

### Phase 1：先把通讯录详情页和聊天信息侧栏收成一套
- 目标：联系人在通讯录和聊天里看起来是同一个人。
- 要做的事：
  - 抽共享的桌面联系人资料组件
  - 统一字段顺序
  - 统一顶部身份模块
  - 统一管理区动作顺序
  - 把 `朋友圈 / 共同群聊 / 更多资料` 做成标准入口行

### Phase 2：把通讯录页收成微信主心智
- 目标：一进通讯录就像微信电脑端，不像“微信 + 世界角色目录”。
- 要做的事：
  - `标签` 真实落地
  - `世界角色` 下沉，不再污染微信标准入口组
  - 右侧详情改成紧凑联系人资料页
  - 联系人字段改成逐项编辑或小弹层编辑

### Phase 3：把聊天记录和聊天资料能力做深
- 目标：抄微信电脑端“桌面主端”的查找和资料处理心智。
- 要做的事：
  - 聊天记录搜索服务端化
  - 扩成全量历史而不是当前已加载片段
  - 把聊天文件、图片、视频等入口收进聊天信息侧栏主链路

### Phase 4：最后补真桌面通话和桌面专属能力
- 目标：补齐用户最容易感知到“这不是微信”的重能力缺口。
- 要做的事：
  - 真桌面语音 / 视频
  - 桌面窗口能力
  - 备份 / 存储管理

## 推荐优先级

### 第一优先级
- 通讯录右侧详情微信化
- 聊天信息侧栏与通讯录详情统一
- 标签真实落地
- 朋友圈 / 共同群聊入口接通

### 第二优先级
- 聊天记录搜索做深
- 聊天资料聚合做深
- 菜单顺序与文案微信化
- 整体视觉去项目化

### 第三优先级
- 真桌面通话
- 独立窗口 / 置顶窗口
- 备份 / 存储 / 桌面工具化能力

## 一句话结论
- 现在的桌面端已经不是“功能太少”，而是“微信骨架有了，但通讯录资料页、聊天信息侧栏、聊天检索深度、桌面主端能力还没完全抄到位”。
- 真要做到“完全照微信电脑端来”，先别再补零散按钮，先把 `联系人资料体系` 和 `聊天资料体系` 这两条主轴收口。
