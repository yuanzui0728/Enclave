# Web端手机版消息页与聊天页对齐微信手机端规划

## 目标
- 范围覆盖 `apps/app` Web 端手机版：
  - 消息列表页：`/tabs/chat`
  - 单聊页：`/chat/$conversationId`
  - 群聊页：`/group/$groupId`
  - 聊天详情页：`/chat/$conversationId/details`、`/group/$groupId/details`
- 目标口径不是“参考微信”，而是尽量按微信手机端当前主路径做一比一复刻：
  - 视觉层收平
  - 手势和操作路径贴齐
  - 能做真能力的直接做真能力
  - 暂时做不了的微信专属能力，先按真入口做占位，但不能做成假可用

## 现状结论

### 1. 当前已经有的能力
- 消息列表已经有：
  - 会话拉取
  - 订阅号消息 / 服务号消息入口
  - 顶部搜索框
  - 右上角 `+` 快捷菜单
- 聊天页已经有：
  - 文本发送
  - 表情包发送
  - 图片 / 文件 / 名片 / 位置发送
  - Web 语音转文字输入
  - 输入中提示
  - 聊天背景
- 详情页已经有：
  - 单聊：查找聊天记录、置顶、聊天背景、清空记录、投诉、拉黑
  - 群聊：群公告、置顶、免打扰、保存到通讯录、显示群成员昵称、聊天背景、隐藏聊天、清空记录、退出群聊

### 2. 当前最关键的问题
- 现在不是“功能完全没有”，而是“功能骨架和微信不一致，很多关键操作还没落到移动端主路径里”。
- 当前手机版聊天 UI 还有明显项目自定义风格：
  - 暖色渐变背景
  - 大圆角悬浮感
  - 橙色发送按钮
  - 头部副标题使用“连接顺畅”
- 这些都和微信手机端当前的平灰底、白气泡、绿色发送、弱装饰风格相反。

## 微信手机端基线

### 已核对的公开信息
- 2026-04-10 查询到的微信 App Store 描述显示，微信当前明确支持：
  - 多媒体消息
  - 群聊与通话
  - 语音和视频聊天
  - 表情商店
  - 实时位置共享
  - 消息翻译
  - 自定义聊天墙纸
  - 自定义消息提醒
- 2026-03-26 微信官方经媒体转述再次说明：
  - 微信仍没有“已读”机制
  - `对方正在输入...` 仍是现网行为
- 2025-03 到 2025-10 的近期公开更新里，可以确认微信聊天主路径还在继续增强：
  - 长按消息后可快速多选，支持“选择到这里”
  - 群聊 `@` 和引用链路继续优化
  - 引用消息可选择部分文字
  - 语音发送界面改成更强的滑动手势提示
  - 左滑删除单个聊天后支持 5 秒撤销
  - 聊天页顶部时间标签支持切换完整日期显示

### 对本项目有用的结论
- `没有已读` 不是缺陷，当前项目也不需要加。
- `对方正在输入...` 当前项目已有，属于和微信一致的少数项。
- 真正需要补的是：
  - 消息列表手势
  - 聊天气泡长按菜单
  - 引用回复
  - 语音按住说话
  - 输入区布局与状态切换
  - 会话管理动作进入消息列表，而不是埋在详情页

## 当前代码与微信的差异

### A. 消息列表页差异

#### 1. 会话项缺少微信核心操作
- 当前 `apps/app/src/routes/chat-list-page.tsx` 只有点击进入，没有左滑操作。
- 微信手机端会话列表的核心效率来自左滑与长按：
  - 置顶
  - 标为未读
  - 免打扰
  - 删除 / 不显示
- 现状后果：
  - 置顶、免打扰、隐藏都被埋到详情页或根本没进手机版主路径
  - 消息列表不像“消息管理页”，更像“纯导航页”

#### 2. 会话视觉结构还不像微信
- 当前列表项已有头像、标题、预览、时间、未读数，但还缺微信常见信息层：
  - 免打扰小铃铛
  - 群聊最后一条消息的发送者前缀
  - 草稿 / 提醒 / 系统态标识
  - 左滑后的操作底板
  - 被删除聊天的可撤销提示
- 当前置顶态是暖色背景，不是微信式浅灰区分。

#### 3. 顶部 `+` 快捷菜单不完整
- 当前只有：
  - 发起群聊
  - 添加朋友
  - 扫一扫占位
  - 收付款占位
- 微信手机端常用心智至少还包括：
  - 收付款真实入口
  - 扫一扫真实入口
  - 二维码 / 添加朋友联动
- 这里可以先保留入口顺序和视觉，但未接能力的项必须明确不可用，不要假跳转。

### B. 单聊页差异

#### 1. 页面骨架不对
- 当前单聊页头部是自定义样式：
  - 渐变底
  - 圆角按钮
  - 副标题默认写“连接顺畅”
- 微信单聊页是更克制的：
  - 平灰顶栏
  - 中间标题
  - 几乎无多余文案
  - 右上角直接承接音视频或更多
- “连接顺畅”这类文案不属于微信聊天主界面的信息结构，应去掉。

#### 2. 消息区排版还没贴到微信
- 当前消息区的问题：
  - 背景过暖、层次过多
  - 时间戳几乎每条都露出，密度偏高
  - 系统提示样式和微信不一致
  - 消息高亮样式偏产品自定义
- 微信手机端最近还支持点击时间切换完整日期，这个细节当前完全没有。

#### 3. 长按菜单缺失
- 当前 `apps/app/src/components/chat-message-list.tsx` 的上下文菜单只在 `桌面端 + 群聊` 开启。
- 这意味着移动端单聊和群聊都没有微信最核心的长按能力：
  - 回复
  - 复制
  - 转发
  - 收藏
  - 提醒
  - 多选
  - 撤回
  - 删除
- 这是当前和微信差距最大的一项。

#### 4. 单聊还没有真正的引用回复
- 单聊线程 `apps/app/src/features/chat/conversation-thread-panel.tsx` 目前没有 reply draft 逻辑。
- 后端 / 消息渲染层已经具备引用展示基础，但单聊发送链路没接。
- 结果是：
  - 群聊代码里已经预埋回复结构
  - 单聊完全没有
  - 移动端更是长按入口都没有

#### 5. 输入区交互不对
- 当前移动端输入区是：
  - 左侧 `Mic`
  - 中间输入框
  - 输入框里 `Smile`
  - 空文本时右侧 `+`
  - 有文本时橙色“发送”
- 微信手机端更接近：
  - 左侧是“语音 / 键盘模式切换”
  - 中间输入框或按住说话大按钮
  - 右侧 `表情`、`+`
  - 有文本时是绿色发送按钮
- 当前最大差距：
  - 没有“按住说话”
  - 没有“滑到这里转文字 / 取消”的手势层
  - 发送按钮颜色和形态不对
  - 语音能力现在是“转文字工具”，不是微信式语音输入主流程

#### 6. `+` 面板能力还不够微信
- 现在已经支持：
  - 相册
  - 拍摄
  - 名片
  - 位置
  - 文件
- 但和微信对比还缺：
  - 入口排序更接近微信
  - 视频通话
  - 红包 / 转账 / 礼物类位
  - 收藏
  - 可能的照片 / 实况图扩展
- 这里要分两层：
  - UI 和交互可以先完全抄微信
  - 支付 / RTC 等重能力不能假装已完成

### C. 群聊页差异

#### 1. 群聊比单聊更接近，但仍未进入移动端真可用状态
- `apps/app/src/features/chat/group-chat-thread-panel.tsx` 已经有：
  - 群公告条
  - reply draft
  - `@` mention candidate
  - 群昵称显示开关
- 但 reply 和 `@` 目前都没有变成“移动端微信式操作”：
  - 回复入口仍依赖桌面端上下文菜单
  - mention candidate 只给桌面端
- 所以群聊虽然代码准备更多，但移动端使用体验并没有真的贴近微信。

#### 2. 群聊顶部缺微信式标题细节
- 微信群聊头部常见是：
  - 群名
  - 人数或弱提示
  - 右上角更多
- 当前我们单独插了一条公告带，这点可以保留，但视觉需要更像微信，不要做成品牌提示条。

### D. 聊天详情页差异

#### 1. 单聊详情有一部分是假持久化
- 当前单聊详情里的：
  - `消息免打扰`
  - `强提醒`
- 仍然是前端 `localStorage` 偏好，不是后端真实会话状态。
- 但后端和 contracts 实际已经有 `setConversationMuted` 能力。
- 结论：
  - 这不是“后端缺失”，而是移动端详情页还没接真接口。

#### 2. 微信常用动作没有前置到列表页
- 置顶 / 免打扰 / 隐藏聊天现在更多在详情页里。
- 微信手机端很多高频操作是在消息列表左滑就完成的。
- 所以详情页本身不是第一优先级，应该把动作前移。

## 直接可复用的现有能力

### 已有后端 / contracts，不需要新开接口
- 单聊：
  - 置顶
  - 免打扰
  - 隐藏聊天
  - 清空聊天记录
- 群聊：
  - 置顶
  - 免打扰
  - 隐藏聊天
  - 清空聊天记录
  - 保存到通讯录
  - 群成员昵称显示
- 聊天消息：
  - 图片
  - 文件
  - 名片
  - 位置
  - 表情包
- 结论：
  - 很多微信式能力已经有后端基础，优先做移动端入口和交互，不要再先补接口。

### 当前真正还缺的后端能力
- 会话 `标为未读`
- 微信式移动端长按动作链路需要的：
  - 转发
  - 收藏
  - 删除单条消息
  - 撤回单条消息
  - 多选消息批量处理
- 可能的微信专属扩展：
  - 语音消息气泡
  - 音视频通话
  - 红包 / 转账
  - 实况图完整链路

## 完全抄微信的实施顺序

### Phase 1：先把页面骨架抄对
- 目标：
  - 不增加新业务，只先让用户一眼觉得这是微信手机端聊天页
- 先做内容：
  - 消息列表页改成平灰基底
  - 会话项排版、时间、未读、置顶背景贴近微信
  - 聊天页去掉暖色渐变和“连接顺畅”
  - 输入区按钮顺序、颜色、尺寸对齐微信
  - 详情页视觉继续收平
- 这一步完成后，至少“像微信”。

### Phase 2：把高频操作抄进消息列表和聊天页
- 目标：
  - 让操作路径像微信，而不是只是长得像
- 必做项：
  - 消息列表左滑：
    - 置顶
    - 免打扰
    - 删除 / 隐藏
  - 长按消息动作面板：
    - 回复
    - 复制
    - 转发
    - 收藏
    - 多选
    - 撤回
    - 删除
  - 单聊接入真实引用回复
  - 群聊移动端接入真实回复与 `@`
- 这一步完成后，主路径才算真的进入微信模式。

### Phase 3：把输入区和语音主链路抄到位
- 目标：
  - 不再只是“有语音转文字工具”，而是有微信式输入模式切换
- 必做项：
  - 左侧做“语音 / 键盘”切换
  - 文本模式和按住说话模式切换
  - 长按说话
  - 上滑取消
  - 滑到这里转文字
  - 表情和 `+` 面板互斥
  - `+` 面板顺序、分页、图标节奏贴微信
- 这一步完成后，输入体验才真正对齐。

### Phase 4：补微信近一年新增的细节能力
- 目标：
  - 把“现在微信已经有、我们还没有”的细节补齐
- 候选项：
  - 引用部分文字
  - 快速多选“选择到这里”
  - 顶部时间标签切换完整日期
  - 左滑删除 5 秒撤销
  - 实况图发送
  - 一次撤回本次连续发送的多条消息
- 这些不是第一天就必须做，但如果目标真是“完全抄”，最终要补上。

## 验收标准
- 用户第一次打开 `/tabs/chat`，不会再看到当前项目的暖色品牌消息页，而会直接感到像微信手机端。
- 用户不进详情页，也能在消息列表完成：
  - 置顶
  - 免打扰
  - 删除 / 隐藏
- 用户在单聊或群聊中长按任何消息，都能拉起微信式动作菜单。
- 单聊和群聊都支持引用回复，且移动端可用。
- 输入区支持微信式文本 / 语音模式切换，而不是只有语音转文字工具。
- 详情页只承接低频设置，不再承载高频会话管理动作。
