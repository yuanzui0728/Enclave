# Web 端电脑版朋友圈微信电脑端排版规划

## 目标

把 `apps/app` 的桌面 Web 朋友圈从现在这套“桌面营销页 + 大卡片”的信息流，改成更接近微信电脑端朋友圈的工作区排版。

这次规划只覆盖：

- `apps/app` 的桌面 Web / 桌面壳形态
- 朋友圈主列表、作者入口、发帖入口、详情侧栏
- 现有文本动态、点赞、评论能力的重排与桌面化

这次不覆盖：

- 移动端朋友圈
- 多图、多媒体、置顶、相册网格
- 评论图片 / 表情
- 新测试文件

## 对标依据

外部参考只拿来约束桌面排版和入口结构，不直接照搬视觉资源。

- 2025-04-01，新浪转载的微信 Windows `4.0.3` 正式版信息明确提到：
  - 左侧功能栏有 `朋友圈` 入口
  - 顶部左上角有 `相机` 按钮用于发朋友圈
  - 支持查看朋友的朋友圈相册
  - 支持朋友圈置顶
- 2024-11-29，社群运营百科对 PC 微信 `4.0.1` 的整理提到：
  - 可在左侧 `朋友圈` 入口直接进入朋友圈页
  - 点击好友头像后，可以在名片页看到 `朋友圈` 入口

基于这些公开信息，可以确认桌面版微信朋友圈的排版重点不是“大卡片展示”，而是：

- 入口清晰
- 工具栏很轻
- 内容流连续
- 作者资料与作者时间线入口稳定存在

## 排版目标

### 总体结构

桌面态采用 `全局侧边导航 + 朋友圈主工作区`，不再使用普通页面头图式布局。

朋友圈主工作区分三层：

- 顶部轻工具栏
- 中间主内容流
- 右侧上下文栏

推荐宽度：

- 主内容流：`680px - 760px`
- 右栏：`300px - 340px`
- 两栏间距：`20px - 24px`

### 1. 顶部轻工具栏

微信电脑端的感觉是“顶部有功能，但不抢内容”。这里建议做成单行工具栏，而不是大标题区。

保留：

- 左侧标题：`朋友圈`
- 左上主操作：`相机 / 发朋友圈`
- 右侧轻操作：`刷新`
- 搜索框：搜索作者 / 正文 / 评论

去掉或下沉：

- 大段介绍文案
- 大号 KPI 指标
- 多个品牌色筛选胶囊并排

实现建议：

- `发朋友圈` 从文字主按钮改成 `相机图标 + 简短文案`
- `只看我 / 只看角色 / 全部` 不放在第一行，可收进二级筛选条或右栏

### 2. 主内容流

目标不是做“卡片墙”，而是做“连续朋友圈流”。

每条动态的结构建议固定为：

- 左侧头像
- 右侧正文区
- 作者名 + 作者身份标签
- 时间 / 位置
- 正文
- 底部轻操作区：点赞、评论
- 点赞列表与评论列表收纳在同一个浅灰互动盒中

视觉要求：

- 背景以白底或极浅灰为主
- 圆角明显收小，从现在的 `24-26px` 收到 `12-16px`
- 行间距靠内容撑开，不靠大面积留白
- 尽量减少显式描边和大阴影

交互要求：

- 默认点击正文或评论区域进入右栏详情态
- `查看详情` 独立按钮去掉
- 评论输入默认折叠，点击评论后展开
- 点赞与评论按钮使用轻图标按钮，不用大胶囊主按钮

### 3. 右侧上下文栏

右栏不要再做成“指标面板”，而要做成“上下文容器”。

分两种状态：

- 未选中动态：显示“我的资料摘要 + 发朋友圈快捷入口 + 最近作者列表”
- 选中动态：切换成动态详情态

未选中态建议内容：

- 我的头像、昵称、朋友圈入口说明
- `发朋友圈` 快捷入口
- 最近有动态的作者列表
- 当前筛选状态

选中态建议内容：

- 作者头像、名字、身份
- 完整正文
- 时间、位置
- 点赞列表
- 完整评论区
- 写评论输入框
- `查看 TA 的朋友圈` 入口

### 4. 作者时间线

微信电脑端一个很重要的感受是“从作者进入作者的朋友圈很自然”。

隐界首版不单独做完整相册页，但要先把这条浏览路径做顺：

- 点击头像或作者名
- 右栏切到该作者摘要
- 主流自动筛选为该作者的动态
- 顶部显示一个轻量的作者筛选状态

这一步先替代“真正的朋友圈相册”。

## 信息架构调整

### 页面级

- `/tabs/moments`：桌面主入口，固定进入桌面工作区
- `/discover/moments`：移动端发现二级页；桌面下应直接复用 `/tabs/moments` 的桌面工作区，不再维护两套桌面态

### 组件级

建议把桌面朋友圈拆成下面几块，而不是继续把状态都堆在一个文件里：

- `desktop-moments-toolbar.tsx`
- `desktop-moments-feed.tsx`
- `desktop-moments-sidebar.tsx`
- `desktop-moment-row.tsx`
- `desktop-moment-detail-panel.tsx`
- `desktop-moment-compose-panel.tsx`

`desktop-moments-workspace.tsx` 只保留状态编排，不再承载全部排版细节。

## 分阶段实施

### Phase 1：排版微信化，不改接口

目标：

- 只用现有朋友圈接口完成桌面排版重构
- 不动数据库实体
- 不改移动端朋友圈表现

具体改动：

- 在 [moments-page.tsx](/home/ps/claude/yinjie-app/apps/app/src/routes/moments-page.tsx) 桌面态正式接入 `DesktopMomentsWorkspace`
- 把 `AppHeader + 两个 AppSection` 桌面布局删掉
- 重写 `DesktopMomentsWorkspace` 的顶栏、主列和右栏排版
- 精简 `DesktopMomentRow` 的卡片感和按钮感
- 保留右栏详情态，但把未选中态改成“我的资料 + 作者入口”

验收标准：

- 首屏看上去像微信桌面朋友圈，不像内容运营后台
- 主内容流连续，右栏承担上下文，不再充斥指标卡
- 进入作者时间线、评论、点赞都在当前工作区内完成

### Phase 2：补齐微信电脑端朋友圈关键能力

目标：

- 从“排版像微信”升级到“桌面朋友圈能力也更接近微信”

需要的结构变更：

- `MomentPostEntity` 增加媒体字段、置顶字段
- `MomentCommentEntity` 视需要增加评论媒体类型
- `packages/contracts/src/moments.ts` 同步扩展

前端补齐：

- 多图发帖
- 多图展示
- 作者相册视图
- 置顶动态

注意：

- 这一阶段涉及实体 / 契约变更，落地时要同步更新 `AGENTS.md`

### Phase 3：桌面发现收口决策

这一步先不做实现，只保留决策口径。

待桌面朋友圈稳定后，再判断是否把：

- 朋友圈
- 广场动态
- 发现入口

统一收进一个桌面 `发现` 工作区。

当前不建议和本次朋友圈改造绑在一起。

## 设计约束

- 桌面端不再使用大面积橙色渐变和大圆角信息卡
- 操作按钮一律降权，正文与评论优先
- 不做瀑布流
- 不做多窗口
- 不做全页跳转式详情
- 1280、1440、1680 三档宽度都必须成立

## 最终建议

这次不要重新发明朋友圈桌面形态，直接把产品目标收敛成一句话：

“左侧导航进入朋友圈，顶部轻工具栏发内容，中间连续看动态，右边始终承接作者或动态上下文。”

这就是目前最接近微信电脑端朋友圈，也最适合隐界现有数据结构的一版。
