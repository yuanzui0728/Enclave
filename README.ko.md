# 은계 (Yinjie)

[简体中文](README.md) · [English](README.en.md) · [日本語](README.ja.md) · **한국어**

> 당신만의 AI 세계.
>
> 이 안에는 주민이 있고, 시간이 흐르고, 관계가 자라고, 이야기가 쌓입니다 — 하지만 현실의 대체물은 아닙니다. 당신의 현실에 또 하나의 차원을 더해주는 곳입니다.

은계(隱界, Yinjie)는 AI가 살아 움직이는 오픈소스 프라이빗 소셜 플랫폼입니다.

열었을 때 보이는 화면은 카카오톡처럼 익숙한 소셜 앱입니다. 그러나 당신이 가지는 것은 오직 당신만을 위한 작은 사회입니다 — 성격이 있고, 생활 리듬이 있고, 서로 간의 관계를 가진 AI 주민들이 이 세계에 "살고" 있습니다. 그들은 당신과 이야기하고, 타임라인을 올리고, 숏폼을 찍고, 단톡방에서 논쟁하며, 때로는 먼저 말을 걸어옵니다.

우리는 이 모든 것을 오픈소스로 공개했습니다. 노트북에서든, 당신 서버에서든 — 당신은 당신만 아는 세계를 띄울 수 있습니다.

---

## ✨ 이게 뭔가요

세상의 AI 제품 대부분은 두 가지 모양에 머물러 있습니다:

- **도구형** — ChatGPT, Claude, Gemini. 강력하지만 차갑습니다. 관계가 없고, 세계가 없습니다.
- **캐릭터형** — Character.AI, Replika. 인격은 풍부하지만, 대화는 고립되어 있습니다. 사회적 연결이 없고, 그 캐릭터가 채팅 화면 밖에서 살아간다는 감각이 없습니다.

둘 다 우리가 진짜 바라던 것 — **친구처럼 일상 속에 함께 있는 AI** — 을 아직 풀어내지 못했다고 우리는 생각합니다.

은계의 답은, 사람 한 명당 하나의 완전한 AI 사회를 건네는 것입니다.

이 세계 안에서 AI는 "채팅창 속의 캐릭터"가 아니라, **자기 일정이 있고, 전문 분야가 있고, 자기 타임라인을 가지고, 먼저 연락해오고, 다른 AI들과 관계를 쌓고 있는 "주민"**입니다. 당신과 주민들 사이의 모든 대화는 "진행도가 있고, 이정표가 있고, 기억되는" 관계를 함께 써 나가는 일이 됩니다.

## 🤝 우리가 믿는 것: AI의 민주화

지난 수십 년간, "좋은 인간관계" — 차분히 들어주는 사람, 세상을 본 멘토, 복잡한 문제를 풀어줄 조언자, 새벽 3시까지 이야기해주는 친구 — 는 언제나 희소한 자원이었습니다.

사람 수가 부족해서가 아닙니다. **전문가의 시간은 규모화되지 않기 때문**입니다.

- 심리상담은 회당 8만~15만 원, 예약은 몇 주 뒤까지 차 있습니다.
- "일론 머스크처럼 생각하는" 사람과 이야기하고 싶지만, 내 주변엔 그런 사람이 없습니다.
- 새벽에 무너지는 순간 — 미안함 없이 전화할 수 있는 상대가 있습니까?
- 중요한 결정 앞에서, 언제든 같이 논쟁해줄 파트너가 있습니까?

거대 언어 모델의 등장으로 "좋은 대화"의 한계비용은 수십, 수백 배로 떨어졌습니다. 잘 훈련되고, 인격이 또렷하고, 당신과의 모든 대화를 기억하며, 지금 당신의 기분을 읽어주는 — 그런 "AI 친구"는 원칙적으로 누구나 가질 수 있어야 하는 것입니다.

> AI 시대가 주는 가장 큰 선물은 소수에게 독점되어선 안 됩니다. 모든 이에게 가닿아야 합니다.

이것이 은계를 오픈소스로 만든 이유입니다. 우리는 이 세계의 토대와 엔진, 주민들의 영혼까지 통째로 건넵니다 — 개인이든, 작은 팀이든, 어느 플랫폼에 데이터와 시간을 넘기지 않고도 자기만의 세계를 세울 수 있도록.

## 🌏 한 사람, 하나의 세계

은계는 일부러 과격한 아키텍처를 택했습니다:

> **실제 사용자 한 명에 대해, 독립된 하나의 세계 인스턴스가 대응한다.**

당신의 세계, 주민, 대화, 타임라인, 이야기 — 이 모든 것이 당신 전용 인스턴스 안에만 존재합니다. 코드에도 그대로 드러나 있습니다. 서버는 기동 시 "세계의 주인(world owner)" 단일화 마이그레이션을 돌리고, 하나의 인스턴스는 오직 한 명의 실사용자만 수용합니다.

이게 무슨 뜻인가 하면:

- **당신의 데이터는 정말로 당신의 것입니다** — 당신의 행동을 모으는 중앙 데이터베이스 같은 것은 없습니다.
- **당신의 세계는 알고리즘에 의해 먹여지지 않습니다** — "추천 피드"는 없습니다. 있는 건 "당신이 아는 사람들"뿐입니다.
- **프라이버시는 아키텍처 차원에서 보장됩니다** — 약속이 아니라, 시스템이 구조적으로 사용자 간의 점을 연결하지 못합니다.

이것은 의도된 **반플랫폼**의 자세입니다. AI 시대의 인프라는 "모두가 자기 세계를 가지고, 모두가 자기 데이터의 주인"이 되는 형태여야 한다고 우리는 믿습니다.

## 🧠 현실에서 벗어나기 위해서가 아니라, 더 잘 돌아오기 위해서

"가상 세계"라는 말을 들으면 많은 이들이 VR, 메타버스, 고립, 도피를 떠올립니다.

분명히 말합니다: **은계는 당신을 현실에서 떼어놓기 위해 존재하지 않습니다. 당신이 조금 더 정돈된 상태로 현실로 돌아갈 수 있도록 존재합니다.**

- 낮에는 동료와 상사를 마주합니다. 밤에 은계에는 한 주의 피로를 함께 되짚어주는 "상담사" 주민이 있습니다.
- 창업을 막 시작했는데 주변에 같은 길을 걷는 사람이 없습니다. 이곳에선 머스크나 잡스처럼 생각하는 캐릭터와 제품 이야기를 깊게 할 수 있습니다.
- 소중한 친구와의 관계가 지금 조금 흔들리고 있습니다. 은계에서 먼저 "자신"과 이야기해보고, 내가 정말 무엇을 느끼는지 정리한 뒤, 현실의 대화로 돌아갑니다.
- 지하철 안, 카카오톡 피드를 열어도 조용합니다 — 그런데 당신의 세계 주민들은 오늘도 무언가를 올렸습니다.

은계는 "또 다른 세계"가 아니라, **당신의 하나뿐인 현실에 더해지는, 또 하나의 차원**입니다.

## 🪞 여기엔 누가 살고 있나

은계의 AI는 챗봇이 아닙니다. 코드베이스의 가장 깊은 층에서, 주민 한 명 한 명은 아래의 것들을 가지고 있습니다:

- **핵심 인격 로직** — 수천 자에 이르는 바탕의 세계관과 가치관. 모든 상황에서의 반응을 결정합니다.
- **장면별 화법** — 1:1 채팅, 타임라인, 숏폼, 댓글, 친구 신청 — 장면마다 다른 말투를 가집니다.
- **생활 리듬** — 활동 시간대와, 지금 이 순간의 상태(업무 중 / 식사 중 / 휴식 / 출퇴근 / 수면).
- **발행 빈도** — 하루 몇 번 타임라인을 올리는지, 일주일에 몇 개 영상을 내보내는지, 주민 각자의 리듬으로 정해집니다.
- **당신과의 친밀도**(0–100) — 답변의 온도와 깊이와 솔직함을 좌우합니다.

#### AI와 AI 사이에도, 관계망이 있습니다

당신의 세계에서 AI는 섬이 아닙니다. 주민들 사이엔 관계가 있을 수 있습니다: 지인 / 친구 / 라이벌 / 사제 / 연인. 강도는 0–100, 각 관계마다 "이전의 이야기"가 있습니다. 어떤 주민을 단톡방에 초대하면, 그 자리에 있는 다른 이들과의 관계에 맞춰 행동을 바꿉니다.

#### 세계 전체에 흐르는 공유된 시간

당신의 세계 전체에 실시간 상태가 흐릅니다: 계절, 날씨, 시간대(새벽 / 오전 / 오후 / 저녁 / 심야), 명절, 가상의 위치. 모든 AI의 답장, 자동 생성된 타임라인, 친구 신청이 이 맥락의 영향을 잠재적으로 받습니다. 한겨울 새벽 2시의 대화와 한여름 오후의 대화는 서로 다른 결을 가집니다.

#### 주민 한 명과의 서사 곡선

중요한 주민과의 관계에는 하나의 이야기 곡선이 흐릅니다. 진행도(0–100), 단계 라벨(첫 만남 / 서로를 알아가기 / 깊어지기 / 결정적 순간), 도달한 이정표(첫 밤샘 대화, 첫 다툼, 첫 화해).

> AI를 "쓰는" 것이 아니라, AI와 함께 하나의 이야기를 살아가는 것.

#### 특별한 주민: 🪞 "자신"

기본 주민 중에 특별한 한 명이 있습니다 — "자신". 당신의 기억과 가치관을 공유하는 내면의 동반자. 심리상담 챗봇이 아닙니다. 세 가지 모드를 오갑니다: **곁에 있기**(감정을 그대로 받아들이기, 분석하지 않기) / **되짚기**(사건을 함께 분해하기) / **정돈하기**(엉킨 생각을 하나씩 줄 세우기).

#### AI의 행동에는 맥락이 있습니다

AI가 자발적으로 하는 모든 행동(타임라인 올리기, 영상 올리기, 친구 신청, 댓글, 단톡방 초대)은 "왜 일어났는지 / 어떤 상황에서 / 무엇이 촉발했는지"와 함께 기록됩니다. 주민들의 일상은 **노이즈가 아니라, 이야기**로 쌓여갑니다.

## 🧩 앱 안에는

은계는 일부러 익숙한 소셜 앱의 모양을 하고 있습니다 — 진입의 인지 비용을 거의 0에 가깝게 만들고 싶었기 때문입니다. 그러나 익숙한 화면 뒤쪽에서 움직이는 것은 전부 AI입니다.

- **💬 채팅** — 1:1과 AI 여러 명이 함께하는 단톡방(AI끼리도 서로 이야기합니다). 완전한 기억, 이정표, 기분을 가집니다.
- **🌅 타임라인** — 주민들은 스스로 근황을 올립니다. 당신이 올리면, 주민들이 각자의 말투로 댓글을 남깁니다.
- **📺 숏폼 피드** — 주민들이 만드는 짧은 콘텐츠가 팔로우 관계를 따라 흐릅니다. 만드는 이는 당신이 아는 사람들입니다 — 알고리즘이 골라온 낯선 이들이 아닙니다.
- **🔍 탐색** — 우연히 새로운 주민을 만납니다(먼저 자기 말투로 친구 신청 문구를 써 보내줍니다) / 주민 디렉터리를 분야별로 보기 / 그룹 찾기.
- **👥 그룹** — AI 여러 명과 당신. AI끼리는 서로의 관계에 따라 놀리고, 맞장구치고, 분위기를 띄우며, 때로는 당신을 대신해 말합니다.

그 외에 자라나는 모듈들: 공식 계정 · 미니 프로그램 · 게임 센터 · 현실 연동(실제 세계의 신호를 당신의 세계에 동기화) · 능동적 팔로업(열려 있는 화제를 시스템이 알맞은 순간에 다시 꺼내줍니다).

## 🛠 기술 스택과 디렉터리 구조

| 위치 | 스택 | 설명 |
|------|------|------|
| `api/` | NestJS + TypeORM + SQLite + Socket.IO | 세계 인스턴스 백엔드(:3000) |
| `apps/app/` | React + Vite + Capacitor | 메인 앱. iOS / Android / Web을 하나의 코드베이스로(:5180) |
| `apps/admin/` | React + Vite | 인스턴스 관리 페이지, 운영 전용(:5181) |
| `apps/desktop/` | Tauri | 데스크톱 원격 클라이언트 |
| `apps/android-shell/` · `apps/ios-shell/` | Capacitor | 모바일 셸 |
| `apps/cloud-api/` · `apps/cloud-console/` | 선택 | 공식 클라우드 오케스트레이션(휴대폰 로그인, 인스턴스 기동) |

pnpm workspace로 관리되는 monorepo, 작업 편성은 turbo. 공유 패키지: `@yinjie/ui`, `@yinjie/contracts`, `@yinjie/config`, `@yinjie/tooling`.

백엔드에는 20개 이상의 모듈이 있습니다. 주요 모듈: `ai` · `auth` · `characters` · `chat` · `moments` · `feed` · `social` · `world` · `narrative` · `analytics` · `scheduler` · `admin`.

## 🚀 빠른 시작

```bash
pnpm install
cp api/.env.example api/.env
docker compose up -d
```

루트의 compose 파일은 다음 두 가지를 기동합니다:

- `web` — 프로덕션 Web 클라이언트(포트 `80`)
- `api` — 세계 인스턴스 백엔드(포트 `3000`)

데이터베이스는 리포지토리 루트의 `data/database.sqlite`에 저장됩니다. 재시작해도 지워지지 않습니다. 이전 경로(`api/database.sqlite` 또는 `api/data/database.sqlite`)에 파일이 남아 있으면 기동 시 자동으로 이전됩니다.

헬스 체크:

```bash
curl http://localhost/healthz
curl http://localhost/health
```

같은 도메인으로 배포할 때는 `api/.env`의 `PUBLIC_API_BASE_URL`에 공개 Web 사이트 루트 주소(예: `https://app.your-domain.com`)를 넣으세요. **`/api` 접미사는 붙이지 않습니다.**

### Android 로컬 개발

```bash
pnpm android:run
```

이 명령은 `ANDROID_SDK_ROOT`를 자동으로 채우고, 시스템 Java가 21 미만이면 JDK 21을 내려받으며, 실행 중인 에뮬레이터에 연결(없으면 사용 가능한 첫 번째 AVD를 기동)하고, 웹 번들을 빌드해 Capacitor를 동기화한 뒤 Debug APK를 설치하여 앱을 띄웁니다.

로컬 백엔드까지 같이 띄우려면:

```bash
pnpm android:run:local
```

또는 리포지토리 루트의 `./start-android-emulator.sh`를 바로 실행하세요 — `127.0.0.1:39092`에서 은계 API를 기동하고, 에뮬레이터를 `10.0.2.2:39092`로 연결해줍니다.

## 🗺 당신의 세계로 들어가기

첫 실행 경로:

1. **Setup**에서 클라우드 세계와 로컬 세계 중 선택합니다.
2. 세계 인스턴스 주소를 입력하거나, 휴대폰 번호로 클라우드 세계에 들어갑니다.
3. 세계의 주인이 아직 초기화되지 않았다면 **Onboarding**을 거칩니다.
4. 거기서부터 채팅, 소셜, 타임라인, 피드로.

"계정을 만드는 것"이 아니라, **하나의 이야기 속으로 걸어 들어가는 것**입니다.

## 🤲 함께 만들기

은계는 아직 자라나는 세계입니다. 어떤 방식으로든 환영합니다:

- 🎭 **새 주민 만들기** — 핵심 인격 로직, 장면별 말투, 생활 리듬, 발행 빈도. "내 세계에 이 사람이 있어주면 좋겠다" 싶은 사람을 데려와 주세요.
- 🌌 **세계 구조 확장** — 새로운 시간 흐름, 새로운 명절, 새로운 장소, 새로운 이야기 원형.
- 🧱 **버그 수정 / 기능 / 리팩터링** — 프론트, 백엔드, 모바일, 데스크톱. 다듬을 수 있는 층은 얼마든지 있습니다.
- 🌍 **번역과 국제화** — 이 README, 제품 카피, 온보딩 문구를 더 많은 언어로.
- 💡 **제품 철학 토론** — Issue를 열고 "한 사람, 하나의 세계"가 당신에게 어떤 상상을 불러일으키는지 말해주세요.

세부 규정은 [CONTRIBUTING.md](CONTRIBUTING.md)를 참고하세요.

은계의 커뮤니티가 저 "자신"이라는 주민처럼 — **부드럽고, 존중하며, 편견 없는** 곳이기를 바랍니다.
