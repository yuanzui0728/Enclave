# Mobile Web Fix Plan

日期：2026-04-20
范围：手机浏览器 Web 端（iOS Safari、Android Chrome）

## 本轮已确认问题

### P0

- 路由恢复白名单漏掉 `"/tabs/moments"`、`"/tabs/feed"`、`"/tabs/channels"`、`"/tabs/games"`、`"/tabs/mini-programs"`。
- 结果：这些页面在移动 Web 中可正常进入，但刷新、冷启动或浏览器回收后，启动页不会恢复到上次停留位置。
- `"/tabs/favorites"` 和 `"/notes"` 虽然可达，但移动布局本身不完整，不应作为手机冷启动恢复目标。
- 结果：如果把它们一并加入恢复白名单，手机用户可能被带回“仅桌面可用”的限制页。

### P1

- 启动页内联样式仍使用 `100vh`，没有给 `100dvh` 和 Safe Area 做兜底。
- 结果：移动浏览器地址栏展开/收起时，首屏可能出现高度抖动、底部留白或内容轻微跳动。
- `/tabs/*` 二级页沿用了“只适用于四大主 tab”的底部导航显示条件。
- 结果：`收藏 / 朋友圈 / 广场动态 / 视频号 / 游戏 / 小程序` 等页面会出现一个没有高亮项的底栏，占空间且容易误导用户。
- 聊天中的 `note_card` 在手机端仍跳转 `"/notes"`。
- 结果：该页面实际是桌面工作区，移动浏览器会进入受限布局，无法完成正常阅读。
- 角色详情返回态没有覆盖若干手机高频入口。
- 结果：用户从 `星标朋友 / 标签 / 世界角色 / 聊天信息 / 聊天头像或名片 / 角色朋友圈` 进入角色详情后，如果刷新、冷启动或浏览器回收，再点击返回，容易退回默认 `"/tabs/contacts"`，丢失原上下文。

## 本轮已完成处理

- 扩充移动 Web 路由恢复白名单，覆盖当前真实可达且适合手机恢复的页面。
- 将启动页高度从单一 `100vh` 扩展为 `100vh + 100dvh`，并补上 Safe Area padding。
- 将移动端底部 tab 的显示范围收紧到四个主 tab，避免二级页误挂底栏。
- 给 React Suspense 启动占位页补上 `100dvh` 和 Safe Area padding。
- 将聊天里的笔记卡片改为手机端原地摘要预览，补上分享/复制和定位消息入口，避免再跳桌面专用 `"/notes"`。
- 给角色详情补上来源页返回态，覆盖 `星标朋友 / 标签 / 世界角色 / 聊天信息 / 聊天头像或名片 / 角色朋友圈` 进入链路，刷新或冷启动后仍能回到原页面。
- 给移动端 `角色详情` 空状态卡片补上来源页回退；当角色资料缺失或刷新后落到空状态时，按钮会优先回原来源页，不再固定退回 `"/tabs/contacts"`。
- 给移动端 `角色详情 <-> 角色朋友圈` 之间的互跳补上 `returnPath/returnHash`，避免这条链路在刷新后退回默认页。
- 给移动端朋友圈主列表补上“进入角色朋友圈”入口，并把返回态接回 `/tabs/moments` 或 `/discover/moments`。
- 给移动端 `角色朋友圈` 空状态页补上来源页回退；当角色资料缺失或刷新后落到空状态时，主按钮会优先回原来源页，其次回角色资料，不再固定跳 `"/discover/moments"`。
- 给移动端 `视频号 -> 作者主页` 补上来源页返回态，并把来源 `section/post` 一起带回去，避免从 `推荐 / 朋友 / 关注 / 直播` 进入作者页后，刷新再返回时丢到默认推荐流。
- 给移动端 `视频号` 首页状态卡补上来源页回退；读取失败或暂时空白时，主按钮会优先回原来源页，没有来源态时再继续重新加载或生成内容，不再只能依赖页头返回。
- 给移动端 `公众号列表 / 公众号详情 / 文章 / 订阅号消息 / 服务号会话` 补上统一 `returnPath/returnHash`，把 `列表 -> 详情 -> 文章` 和 `消息 -> 详情/文章` 的回程链路接通，避免刷新后退回默认联系人或消息首页。
- 给移动端 `服务号会话` 组件自身的返回兜底补上 `returnPath/returnHash`；从带来源态的服务号会话在刷新或冷启动后直接返回时，会优先回到原来源页，不再固定掉回 `"/tabs/chat"`。
- 给移动端 `公众号主页 / 公众号文章` 的错误态补上来源页回退；当刷新、冷启动或读取失败落到状态卡时，主按钮会优先回原来源页，其次回公众号主页或列表，不再只能依赖页头返回。
- 给移动端 `订阅号消息 / 服务号会话` 的状态卡补上来源页回退；读取失败或暂时空白时，主按钮会优先回原来源页，其次打开公众号列表或公众号主页，不再只剩页头返回。
- 给移动端 `公众号列表` 的状态卡补上来源页回退；读取失败或搜索无结果时，主按钮会优先回原来源页，没有来源态时再继续重新加载、清空搜索或打开订阅号消息，不再只剩页头返回。
- 给移动端 `发现 -> 摇一摇 / 场景相遇` 补上 `returnPath/returnHash`；从发现页进入这两个轻工具页后，刷新或冷启动再返回时，会优先回到原发现页，不再固定掉回默认发现首页。
- 给移动端 `发现 -> 朋友圈 / 广场动态` 补上 `returnPath/returnHash`；从发现页进入朋友圈或广场动态后，刷新或冷启动再返回时，会优先回到原发现页，不再固定掉回默认发现首页。
- 给移动端 `朋友圈 / 广场动态` 的状态卡补上来源页回退；读取失败或暂时空白时，主按钮会优先回原来源页，没有来源态时再继续重新加载或发布内容，不再只能依赖页头返回。
- 给移动端 `发现 -> 游戏` 补上 `returnPath/returnHash`；从发现页进入游戏后，当前游戏切换、刷新或冷启动再返回时，会优先回到原发现页，不再固定掉回默认发现首页。
- 给移动端 `游戏 -> 单聊 / 群聊` 的组局邀约会话入口补上外层 `returnPath/returnHash` 透传；从带来源态的游戏页把组局邀约投递到会话后，再返回游戏页、再返回外层来源页时，不会在游戏页这层把发现或搜索来源吃掉。
- 给移动端 `发现 -> 视频号` 补上 `returnPath/returnHash`；从发现页进入视频号后，切 section、刷新或冷启动再返回时，会优先回到原发现页，不再固定掉回默认发现首页。
- 给移动端 `发现 -> 小程序` 补上 `returnPath/returnHash`；从发现页进入小程序后，当前工作台切换、刷新或冷启动再返回时，会优先回到原发现页，不再固定掉回默认发现首页。
- 给移动端 `游戏 / 小程序` 当前选中工作台补上 URL search 同步，刷新或冷启动后可以恢复到当前选中的游戏或小程序，而不是回到默认项。
- 给移动端 `群接龙来源 -> 小程序` 补上返回兜底；当从群聊里的接龙卡片进入小程序后，刷新或冷启动再返回时，会优先回到来源群聊，而不是固定跳 `"/tabs/discover"`。
- 给移动端 `搜索 -> 朋友圈 / 广场动态` 补上来源页 `returnPath/returnHash`；从搜索结果进入 `"/discover/moments"` 或 `"/discover/feed"` 后，刷新或冷启动再返回时，会优先回到原搜索页并保留搜索词与分类，不再一律掉回发现首页。
- 给移动端 `搜索 -> 角色详情 / 公众号详情 / 公众号文章` 补上来源页 `returnPath/returnHash`；从搜索结果进入这些详情页后，刷新或冷启动再返回时，会优先回到原搜索页并保留原搜索上下文，而不是退回默认联系人或消息页。
- 给移动端 `搜索 -> 小程序` 补上来源页 `returnPath/returnHash`；从搜索结果或最近使用入口进入 `"/discover/mini-programs"` 后，刷新或冷启动再返回时，会优先回到原搜索页并保留搜索上下文，而不是固定退回发现首页；群接龙来源仍优先回群聊。
- 给移动端 `搜索 -> 单聊 / 群聊` 补上来源页 `returnPath/returnHash` 和消息高亮透传；从搜索结果进入会话后，刷新或冷启动再返回时，会优先回到原搜索页并保留搜索上下文，不再统一掉回 `"/tabs/chat"`。
- 给移动端 `搜索里的视频号收藏 -> 视频号` 补上来源页 `returnPath/returnHash`，并让视频号页切 section、进入作者主页再返回时继续保留这份来源态；从搜索结果里的视频号收藏进入后，刷新或冷启动再返回时，会优先回到原搜索页，不再固定退回 `"/tabs/discover"`。
- 给移动端 `单聊 -> 聊天信息 -> 查找聊天记录 / 聊天背景` 补上 `returnPath/returnHash` 透传；从带来源态的单聊进入聊天信息，再进入查找聊天记录或聊天背景后，刷新或冷启动再返回时，仍会优先回原来源页，不再统一掉回 `"/tabs/chat"`。
- 给移动端 `单聊 / 聊天信息 -> 语音或视频通话` 补上原会话 `hash` 透传；从带来源态的单聊或聊天信息进入通话页后，返回聊天、切换语音通话、查看聊天信息时都会继续保留原来的 `returnPath/returnHash`，避免这条链在刷新或回退后退回默认 `"/tabs/chat"`。
- 给移动端 `群聊 -> 语音或视频通话` 补上原群聊 `hash` 透传，并让群通话恢复态继续携带这份群聊来源信息；从带来源态的群聊进入通话页后，返回群聊、结束通话回群聊、查看群聊信息时都会继续保留原来的 `returnPath/returnHash`，避免这条链退回默认 `"/tabs/chat"`。
- 给移动端 `角色详情 / 聊天信息 -> 发消息或语音视频通话` 补上会话 `returnPath/returnHash` 透传；从角色详情发起单聊、语音或视频通话，以及从聊天信息页里的数字人引导进入通话后，返回聊天、查看聊天信息、再返回时都会继续保留原外层来源态，不再在刷新或回退后统一掉回默认 `"/tabs/chat"` 或 `"/tabs/contacts"`。
- 给移动端 `聊天信息 / 群信息 -> 隐藏聊天 / 退出或隐藏群聊` 补上外层 `returnPath/returnHash` 回退；从带来源态的单聊或群聊进入详情页后，执行这些操作或在刷新后发现对象已不存在时，都会优先回到原来源页，不再统一掉回默认 `"/tabs/chat"`。
- 给移动端 `聊天背景 / 查找聊天记录 / 群成员管理 / 群聊名称 / 群公告 / 群背景 / 群二维码 / 群查找聊天记录` 补上对象缺失时的外层 `returnPath/returnHash` 回退；这些二级详情子页在刷新、冷启动或数据缺失后，会优先回到原来源页，而不是统一退回 `"/tabs/chat"`。
- 给移动端 `群成员管理 / 群聊名称 / 群昵称` 的状态卡补上回退按钮；读取失败或暂时空白时，主按钮会返回群聊信息，群聊不存在时再优先回外层来源页或消息列表，不再只剩页头返回。
- 给移动端 `聊天背景 / 群背景` 的状态卡补上来源页回退；读取失败时主按钮会优先回原来源页，没有来源态时再重新加载，对象缺失时则回原来源页或消息列表，不再只剩页头返回。
- 给移动端 `群公告` 的状态卡补上来源页回退；读取失败时主按钮会优先回原来源页，没有来源态时再重新加载，对象缺失时则回原来源页或消息列表，不再只剩页头返回。
- 给移动端 `群邀请 / 群二维码` 的状态卡补上来源页回退；群聊、成员或最近会话读取失败时，主按钮会优先回原来源页，没有来源态时再重新加载；群聊不存在时的自动回退也会优先走原来源页，不再直接吃未过滤的 `returnPath`。
- 给移动端 `小程序` 的空状态卡补上来源页回退；从发现、搜索或群接龙来源进入后，如果“我的小程序”为空或搜索没有结果，主按钮会优先回群聊或原来源页，没有来源态时再继续看当前推荐或清空筛选。
- 给移动端 `聊天信息 / 群信息 / 查找聊天记录` 的来源态补上安全过滤；如果这几页拿到桌面专属 `returnPath`，手机端会忽略掉它，并用净化后的 hash 继续往下透传，避免返回或二跳时误跳桌面工作区。
- 给移动端 `视频号 / 游戏 / 小程序` 的来源态同步补上安全过滤；如果这些页拿到桌面专属 `returnPath`，手机端在切 section、进入作者主页、切工作台或投递组局邀约时会丢弃它，不再继续往下透传到更深层路由。
- 给移动端 `新的朋友 / 世界角色 / 星标朋友 / 标签` 的当前页 hash 同步补上安全过滤；如果这些联系人页拿到桌面专属 `returnPath`，手机端在搜索词同步、互跳、进入角色详情时会丢掉它，只继续透传净化后的移动来源态。
- 给移动端 `朋友圈 / 广场动态 / 角色朋友圈 / 发布页` 的当前页 hash 同步补上安全过滤；如果这些页拿到桌面专属 `returnPath`，手机端在同步当前动态、进入角色朋友圈、进入发布页、再从角色朋友圈进角色详情时会丢掉它，只继续透传净化后的移动来源态。
- 给移动端 `公众号列表 / 订阅号消息 / 公众号主页 / 公众号文章 / 服务号会话` 的当前页 hash 同步补上安全过滤；如果这些页拿到桌面专属 `returnPath`，手机端在列表进详情、详情进文章、服务号会话互跳账号与文章时会丢掉它，只继续透传净化后的移动来源态。
- 给移动端 `发现 -> 朋友圈 / 摇一摇 / 场景相遇 / 广场动态 / 视频号 / 游戏 / 小程序` 的当前页 hash 同步补上安全过滤；如果发现页拿到桌面专属 `returnPath`，手机端在继续打开这些入口时会丢掉它，只继续透传净化后的移动来源态。
- 给移动端 `群成员管理 / 群聊名称 / 群昵称` 的当前页 hash 同步补上安全过滤；如果这些页拿到桌面专属 `returnPath`，手机端在返回群聊信息、保存后回群聊信息时会丢掉它，只继续透传净化后的移动来源态。

- 长列表滚动与回顶。
- 聊天里的 `笔记 / 位置 / 名片` 卡片打开后，关闭、分享/复制、定位消息链路是否正常。

## 建议修复顺序

1. 先修壳层问题：路由恢复、视口高度、安全区。
2. 再修通用交互：键盘、滚动、弹层。
3. 最后补业务回归：聊天、朋友圈、视频号、游戏、小程序。
