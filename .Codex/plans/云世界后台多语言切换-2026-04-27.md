# 云世界后台多语言切换规划

日期：2026-04-27

## 目标

- 云世界管理平台支持简体中文、英文、日语、韩语切换。
- 默认语言为简体中文；用户选择持久化到 `cloud-console` 独立本地偏好。
- 云世界管理平台发起的 API 请求携带当前语言。
- 云世界平台后端错误响应按请求语言返回结构化 `errorCode` / `message`。
- 导出、复制、下载通知等非 DOM 文案跟随当前语言。

## 实施步骤

1. 复用现有 `@yinjie/i18n` 运行时 Provider 和 `LanguageSwitcher`，补齐云控制台专用运行时翻译工具。
2. 在 `apps/cloud-console` 的云 API 客户端注入 `Accept-Language` 与 `X-Yinjie-Locale`，并解析结构化错误响应。
3. 将云控制台错误展示、下载通知、等待会话同步导出 / 复制内容接入当前语言。
4. 在 `apps/cloud-api` 增加 locale 解析、错误翻译和全局异常过滤器，保持无语言头时默认中文。
5. 补充共享 contracts 中的 locale 与错误响应类型。
6. 运行最小验证：云控制台 smoke/i18n 测试、云 API typecheck/build 或相关单测。
7. 提交本次变更。
