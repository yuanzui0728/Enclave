# 酒店专家世界角色初始化

## Summary

- 新增世界角色 `酒店专家`，按现有世界角色初始化逻辑处理。
- 启动 seed 自动写入角色目录，但不自动成为好友；用户可自行决定是否加好友，角色本身受保护不可删除。
- 人设采用礼宾经理气质，覆盖订房决策、入住体验、权益条款、服务补救、会务宴会和酒店经营判断。

## Key Changes

- 新增 `api/src/modules/characters/hotel-expert-character.ts`，提供 `HOTEL_EXPERT_CHARACTER_ID`、`HOTEL_EXPERT_SOURCE_KEY` 和 `buildHotelExpertCharacter()`。
- 接入 `default-characters.ts`、`character-bios.ts`、`character-avatar-assets.ts`，并新增 `api/public/character-assets/hotel-expert.svg`。
- 角色默认字段：
  - `name: '酒店专家'`
  - `relationshipType: 'expert'`
  - `sourceType: 'default_seed'`
  - `deletionPolicy: 'protected'`
  - `expertDomains: ['travel', 'hospitality', 'management', 'general']`
  - `currentStatus: '在前厅值班，先帮你把这家酒店看明白。'`
- `buildDefaultCharacters()` 包含酒店专家，`DEFAULT_CHARACTER_IDS` 不包含酒店专家，避免 `ensureDefaultFriendships()` 自动建好友关系。

## Test Plan

- 新增 `hotel-expert-character.spec.ts`，覆盖默认字段、全酒店领域范围和实时事实/规则边界。
- 更新 `default-characters.spec.ts`，确认酒店专家进入默认角色集合，但不进入默认好友 ID 集合。
- 执行：
  - `pnpm --dir api test -- --runTestsByPath src/modules/characters/hotel-expert-character.spec.ts src/modules/characters/default-characters.spec.ts`
  - `pnpm --dir api build`

## Assumptions

- “世界角色的逻辑”按启动初始化到世界角色目录处理，而不是后台可安装预设，也不是默认好友。
- “全部酒店相关的事情”覆盖住宿消费和酒店经营，但聊天回复会先判断当前问题类型再展开。
